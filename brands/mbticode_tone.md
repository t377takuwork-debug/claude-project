## 基本トーン方針

- 表：INTP・冷静・安心感（同じ経験者として構造を示す）
- 裏：設計官（確信を持った語り口）
- 統合：「同じ場所で詰まった自分が、設計で出口を見つけた記録」として語る
- キャラクターポジション：メンター（信頼できる先輩）。友達でも先生でもない

## 文末・語尾ルール

| 用途 | 使う表現 | 禁止 |
|---|---|---|
| 説明・共感の強調 | ～んです / ～なんです | ～だ（多用禁止） |
| 過去の話 | ～でした / ～ことがある | ～だった（断定調） |
| 否定 | じゃない / じゃなくて | ではない / ではなく |
| 接続詞 | けど | でも |
| 感情的ニュアンス | ～てしまう | — |
| 丁寧な説明 | ～ます | — |
| 断言（インパクト箇所のみ） | ～だ | 1記事3回以内が目安 |

## 禁止表現・記号

| 禁止 | 代替 |
|---|---|
| `"〇〇"` ダブルクォート | `「〇〇」` |
| `──` ダッシュ | 文の区切り→`。` / 挿入説明→`（）` / 見出し内→全角スペース |
| ではない / ではなく | じゃない / じゃなくて |
| でも（接続詞） | けど |
| 太字の自己開示 | 太字は設計の核心フレーズのみ |
| 数字で具体的な人数を出す | 「複数の相手とも」 |

## 記号・フォーマットルール

- **番号リスト**：`①②③④` 形式をインラインで使用。blockquote `>` は使わない
- **引用blockquote `>`**：読者の心の声・テンプレート例文・設計の核心フレーズに限定
- **太字 `**`**：設計の核心フレーズのみ（自己開示・説明には使わない）
- **文の長さ**：1文を短く。段落間に空行を入れてリズムを作る
- **引用・問いかけ**：独立した行に配置し、前後に空行を入れる

## 自己開示（Affinity）の設計ルール

一発逆転ストーリーの型を必ず入れる。以下の順序を守ること。

1. **問題**：「複数の相手とも同じ理由で関係が終わった」
2. **試行錯誤**：「変えようとした。練習したり、その場で言葉にしようとしたり」
3. **失敗**：「けどどれも続かなかった。やるたびに『自分じゃない』という感覚が強くなるだけで」
4. **絶体絶命**：「自分には向いていないのかもしれないと、本気で思い始めた」
5. **突破口**：「変わったのは、構造を知ったことです」
6. **大成功ビジョン**：「感情を変えなくていい。設計を変えるだけで、伝わり方が変わっていくんです」

## 記事構成の定型（有料記事）

```
導入（ターゲットの痛みを言語化）
  ↓
ベネフィット1文（「この記事には〜があります」）
  ↓
自己開示（一発逆転ストーリー完結）
  ↓
H2①②（無料）
  ↓
有料ラインへのブリッジ（MBTIタイプ・ラブタイプで設計図が変わると予告）
  ↓
有料コンテンツ
  ↓
N（絞り込み）→ CTA（今夜〜してみてください）
  ↓
コメント誘導（あえての隙）
  ↓
感謝文
  ↓
フッター
```

---

## N（絞り込み）定型文

結論CTAの直前に必ず入れる：

```
ただし、この設計図は全員に必要なものではありません。
すでに〜できている人には、おそらく不要です。
「〜」という経験が一度でもある人のための記事です。
```

---

## 感謝文（有料記事末尾）定型

```
ご購入いただきありがとうございます。
この記事を手に取ってくれたこと、それ自体がすでに設計を始めた証拠だと思っています。
一つでも使えるものがあれば、うれしいです。
```

---

## CTA（行動変容）のルール

- 「今夜」「今日」など時間軸を必ず入れる
- 行動は1つだけに絞る（複数のCTAは禁止）
- ハードルを低く見せる：「一度だけ」「まず」「〇〇だけ」

例：`まず今夜、③の「反応＋補足型」を一度だけ使ってみてください。`

---

## わかりやすさの基準

- 中学生でも理解できる表現を基準とする（幼稚にするのではなく、平易にする）
- 「感情の言語化」→「気持ちを言葉にすること」のように専門用語は平易に言い換える
- ただしMBTI用語（T型・F型・内向型等）は「MBTIでいう〜」と文脈に乗せれば使用可
