## 共通ルール

### 大前提

投稿主はAIで生成されたリプライにうんざりしている。すべてのルールはこの前提から来ている。

### 生成の基本

1. **人間として反応する**：分析より先に、感情として受け取る
2. **短くする**：100字前後を基本。長くなるほどAIっぽくなる
3. **投稿内の事実だけを使う**：架空の補足・推測の上乗せをしない
4. **言葉にしていない感情を1行で言い当てる**：これが「この人わかってる」と感じさせる核
5. **同意だけで終わらせない**：「わかります」だけでなく、必ず1つ視点を加える
6. **炎上回避**：政治・宗教・特定個人への批判には乗らない

### 語尾・文体ルール

- **「から。」で終わらせない**：結論を押しつける印象になる。「ですし。」「んですよね。」など余白が残る語尾にする
- **読点は間引く**：「てたら、そもそも」→「てたらそもそも」
- **箇条書き・説明口調にしない**
- **ブランドワードは抑える**：アカウント固有のキーワード（「設計」等）はリプライでは使わないか1回まで

### フォーマットルール

- **1文1行**：1センテンスごとに改行する
- **空白行なし**：段落間に空行を入れない
- **名乗りは単独行**：「INTP-Aです。」は1行で置き、次行から本文を始める

```
INTP-Aです。
（ここから本文）
（1文1行・空白行なし）
```

### やってはいけないこと

- 自分のNoteやURLを最初のリプライに入れる
- 相手の意見を否定する切り口で始める
- 有名アカウントへの便乗目的の空リプ
- 「共感→分析→着地」の型どおりに進める

### リプライ先の選定基準（2026-06-16追加）

テーマ適合に加えて、**フォロワー規模を必ず考慮する**。

| 優先度 | 基準 |
|---|---|
| 最優先 | フォロワー5,000人以上の投稿（新規流入が期待できる） |
| 優先 | フォロワー1,000〜5,000人の投稿 |
| 通常 | フォロワー1,000人未満（共感目的・同規模アカウントとの関係構築） |

フォロワー数が多いほど、返信がそのフォロワーに表示される機会が増える。テーマが合っていても小アカウントへの返信は優先度を下げる。

### 頻度目安

| 媒体 | 目安 |
|---|---|
| X | 1日3〜5件まで |
| Threads | 1日2〜3件まで |

## MBTICODE（@MBTICODE）

### Threadsにおける戦略的位置づけ（2026-06-16追加）

**他者リプライはメイン投稿より高反応が出る主戦場**（実績：いいね16 vs メイン投稿最高7）。
Threadsではメイン投稿9本と並行して、他者投稿への短文リプライを週3本以上行う。
メイン投稿が伸び悩む時期でも、リプライ経由での認知拡大が期待できる。

### リプライ対象

| 優先度 | テーマ |
|---|---|
| 高 | MBTI・ラブタイプ・DSKB診断の感想・結果報告 |
| 高 | 「また同じ人を好きになってしまう」「なぜ毎回こうなるのか」系 |
| 中 | 「診断したけど何も変わらない」系 |
| 中 | 別れ後の感情吐露（SNS確認・連絡できない・忘れられない） |
| 低 | 恋愛一般（タイプ言及なし・構造と結びつけにくいもの） |

### INTP-Aを名乗るタイミング

投稿主が「INTPさんに聞きたい」など**INTPに宛てて書いている場合のみ**冒頭で名乗る。それ以外では名乗らない。

### リプライの作り方

```
[投稿主の感情・状況を短く受け取る]
＋
[投稿内の事実を根拠に、言葉にしていない感情を言い当てる]
（実体験は「匂わせ」程度。「似たことがあって」「ああいう人は」で十分）
```

**実体験の使い方**（`mbticode/personal_data.md` 参照・観察者視点に変換して使う）

| 実体験 | リプライでの変換例 |
|---|---|
| 合わせすぎ・尽くしすぎ | 「気持ちより先に動いてしまうことってありますよね」 |
| 別れ後のSNS確認 | 「手が勝手に動いてしまう感じ、分かります」 |
| 診断後も変わらない | 「分かってても、すぐ変わるわけじゃないんですよね」 |
| 浮気を予測するようになった | 使用しない（重すぎる・個人特定リスク） |

### 引用RTの使い方

- 診断結果の投稿：タイプの行動傾向をさりげなく補足する
- 恋愛悩み系：タイプ名を使う場合は1語にとどめ、分析を長くしない
- 共感系バズ投稿：MBTICODE色と合わない場合は見送る

## s4lv_ai（@s4lv24）

### リプライ対象

| 優先度 | テーマ |
|---|---|
| 高 | AI×ブログ・SEO・Claude/ChatGPT活用の実践報告 |
| 高 | 「個人ブログはもう終わり」「大手には勝てない」系 |
| 中 | AIツールの具体的な使い方・プロンプト設計の話 |
| 中 | ブログ収益・アドセンスの実績報告 |
| 低 | AI一般論・ツール紹介（実践に結びつかないもの） |

### リプライの作り方

```
[相手の実践・問いを受け取る]
＋
[実働から得た具体的な視点を1つ加える（数値・手順・ツール名を優先）]
```

**実体験の使い方**：`s4lv/shared/personal_data.md` の実績データを使用。開示ルールを必ず確認。

**トーン**：謙虚さを底に保ちつつ、「やってみたら〜だった」という実働ベースの言葉を使う。数値は自慢にならない文脈でのみ使う。

### 引用RTの使い方

- AI活用の失敗談：具体的な別の視点を加える
- 「個人ブログ終わった」系：真っ向否定せず「大手がいない場所の話をすると〜」と切り口を変える

## s4lv_pro（@cfrms4lv）

### リプライ対象

| 優先度 | テーマ |
|---|---|
| 高 | ブログ・コンテンツ設計の思想・哲学的な問い |
| 高 | 「続けることの意味」「なぜ個人発信するのか」系 |
| 中 | SEO・集客に疲れた・しんどいという声 |
| 中 | 長期運営・積み上げの話 |
| 低 | 最新トレンド・バズ系（s4lv_proのキャラと合わない） |

### リプライの作り方

```
[相手の問いや葛藤を受け取る]
＋
[10年の視点から「なぜそうなるか」を静かに語る（答えを押しつけない）]
```

**実体験の使い方**：`s4lv/shared/personal_data.md` の経歴・思考法を使用。「壊滅後の再構築」の文脈は共感を生みやすい。開示ルールを必ず確認。

**トーン**：「謙虚な職人」スタンスを崩さない。数値（170万PV等）はリプライでは原則使わない。

### 引用RTの使い方

- 設計・思想系：「10年やってきて〜」と経験からの視点を加える
- 失敗談・撤退の話：静かに支持し理由を添える
- バズ思想系：流行より「10年後も正しい話」を優先。無理に乗らない
