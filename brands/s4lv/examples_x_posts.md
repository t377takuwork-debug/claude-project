## フック（1行目）

### ❌ Bad → ⭕ Good 対比

| ❌ Bad | ⭕ Good | 何が違うか |
|---|---|---|
| 「文章力を上げる方法を解説します。」 | 「文章術を学ぶのは、砂漠に高級な看板を立てるのと同じでした。」 | 予告ではなくエグい比喩で現実を描写する。命令・宣言をしないのに読者が「マズい」と気づく |

### フック選定の手順（書き始める前に必ず）

投稿前に「ブックマーク／いいね・RP／リプライ」の**どれを狙うか1つ決める**（混在禁止・絶対禁止事項）。
- ブックマーク狙い → 失敗の情景＋手順の予告（損失回避）
- いいね・RP狙い → エグい比喩×謙虚な自省
- リプライ狙い → 属性を絞った問いかけ（「ひっそりと資産を積み上げている仲間、いますか？」）

## 本文（ライブ感の出し方）

### 記憶のゆらぎ

**❌ Bad（機械的な正確さ）**
```
2020年の深夜1時に、アクセス0を確認しました。
```
**⭕ Good**
```
2020年の、あれは冬だったかな。正確な時間は覚えていないけれど、画面に並んだ「0」が、やけに目に焼き付いている深夜がありました。
```
**何が違うか**: 正確なデータより「あれは冬だったかな」の曖昧さが「本当にあった話」の信憑性を作る。数字の正確さと記憶の曖昧さを逆に配置する。

### 接続詞

**❌ Bad（硬い接続詞）**
```
したがって、需要を確認すべきです。
```
**⭕ Good**
```
なんだろうな。結局、テクニック云々の前に「そこに人がいるか」を確認する。ただそれだけのことだったんです。
```
**何が違うか**: 「したがって」＋「すべき」は講義。「なんだろうな」「結局」は隣で喋っている声。**カジュアルにするのは接続詞・語尾・記憶の語り方だけ**で、主張の核心（手順・戦略）は論理的なまま崩さない。

### 経験談

**❌ Bad（短文羅列のAI感・禁止パターン）**
```
記事を書いた。読まれなかった。やめた。設計を変えた。伸びた。
```
**⭕ Good の条件**: 「何があった」ではなく「どう感じていたか」を軸に、胃が痛む情景を1つ混ぜる。です/ます と だ/である は意図的に混在させる（統一しない）。
```
毎朝アクセス解析を開いては、0の並びを見て静かに閉じる。あの頃は、書くことより「見なかったことにする」方が上手くなっていました。変わったのは、書く前に需要を確認するようになってからだ。
```
