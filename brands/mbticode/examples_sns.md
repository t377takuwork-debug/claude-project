## X投稿

### 見本1: 型α（2行逆転）・タイプなし

**❌ Bad（解説モード・予告終止）**
```
尽くしすぎて「重い」と言われてしまう設計があります。
優しさの問題ではなく、重力が一方向にかかりすぎている話です。
心当たりがある人は要注意ですね。
```
**⭕ Good（実投稿）**
```
頑張るほど、なぜか「重い」と言われる夜があった。

優しさの問題じゃない。重力が、一方向にかかりすぎているだけ。

尽くす側と受け取る側、気づかないうちに固定されてしまうことがある。
```
**何が違うか**: Badは「〜設計があります」「〜の話です」「ですね」の3違反に加え、説明する側の声（解説モード）。Goodは体験した側の声（内声モード）で、1行目が「夜があった」という記憶になっている。**説明を捨てて記憶から始めるのが最大の分岐点。**

### 見本2: 型B（観察）→ タイプ着地

**❌ Bad（タイプ名が冒頭・因果が綺麗すぎ）**
```
INFJは限界まで我慢してから離れていくタイプです。
既読が減って連絡が来なくなったのは、我慢が限界に達したからです。
理由を聞いても教えてくれないでしょう。
```
**⭕ Good（実投稿）**
```
急に既読が減って、そのまま連絡が来なくなったことがある。

理由を聞いても「もう大丈夫」としか返ってこなかった。

限界まで我慢してから、静かにドアを閉めるように離れていく。INFJに近い設計に起こりやすいことの一つだ。
```
**何が違うか**: Goodは「私のことだ」と感じるシーン（既読が減る）が先、タイプ名は核心フレーズの後。「静かにドアを閉めるように」という映像があるから保存される。Badは診断結果の読み上げで、1行目で「自分のタイプじゃない→スクロール」が起きる。

### 見本3: 締めの他者投影（使用制限あり）

**⭕ Good（実投稿・1バッチ1本まで）**
```
甘えたいのに、素っ気ない態度を先に出してしまう。

弱いところを見せたら負けだと思ってしまうところがある。

強がりが、いつのまにか甘えられなさに変わっていく。ラブタイプの「ボス猫」に近いタイプに出やすい。

心当たりある人の顔が浮かびませんか？
```
**注意**: 「心当たりある人の顔が浮かびませんか？」は**1バッチ1本まで**。2本目からは独白型（「〜は別の話なんです。」）・命名型（「それを〇〇と呼ぶことにした。」）等に散らす（cheatsheet締めパターン表）。

## Threads投稿

### 見本4: 完結型・体験匂わせ→タイプ中盤着地

**❌ Bad（Xの圧縮版・説明口調）**
```
尽くす側と受け取る側は配置で決まります。FCPEやFAREはこの傾向が強いです。
最初は感謝されていたのに、だんだん当然扱いになっていくのが特徴です。
このパターンに心当たりがある方は注意が必要です。
```
**⭕ Good（実投稿・280字）**
```
自分もそういう設計だったから言えるけど、尽くす側と受け取る側は性格じゃなくて配置で決まっていくことがある。

急な連絡にもすぐ動いてしまう。頼まれる前に先回りしてしまう。最初は「ありがとう」が返ってきていたのに、続けているうちにそれが「してもらって当然」に変わっていく瞬間があった。

忠犬ハチ公（FCPE）や敏腕マネージャー（FARE）に近い設計は、断るという選択肢が浮かぶ前に体が動いてしまう傾向がある。

尽くすほど相手が「考えなくていい側」に固定されていく。それが、あの「重い」の正体だったんです。
```
**何が違うか**: Goodは①冒頭が体験匂わせ（「自分もそういう設計だったから言えるけど」）②タイプ名は3段落目（中盤）③行動の具体（急な連絡にすぐ動く・先回り）でシーンを見せる④「なんです」は締めの1回だけ。Badは3文とも「〜ます/です」の説明で、200字未満・体験ゼロ・Xとの角度差もない。**ThreadsはXの拡張ではなく「中に入る」別コンテンツ。**

## リプライ・引用RT

### 見本5: 引用RT（視点1つ追加型）

**❌ Bad（復唱＋設計ワード＋て形文末）**
```
言語化しない人っていますよね。わかります。
それって優しさの設計なんだと思って。
私も言語化が苦手で。
```
**⭕ Good（実投稿）**
```
言語化しないのは優しさで、言語化できないのは苦手さ。

似てるようで、この2つは別の話なんですよね。

黙って隣にいられる人ほど、本当に必要な一言は逃さない気がします。
```
**何が違うか**: Goodは元投稿にない**区別**（優しさ／苦手さ）を1つ追加し、「んですよね。」「気がします。」で着地。Badは3違反：①元投稿の復唱で視点追加なし ②リプライでの「設計」使用（アカウントアピール感が出るため禁止）③「〜と思って。」「〜苦手で。」のて形文末。

### リプライで使う語尾・使わない語尾（実例準拠）

| ⭕ 使う | ❌ 使わない |
|---|---|
| 「〜んですよね。」「〜気がします。」「〜だけ。」「〜ですし。」 | 「〜だよな」（砕けすぎ） |
| 「〜切ない。」等の直接断言（強い共感の1行目） | 「〜て。」「〜って。」（て形文末） |
| 「たぶん。」を独立させて置く | 「これ、身に覚えがあって。」（導入不要・本題から入る） |
