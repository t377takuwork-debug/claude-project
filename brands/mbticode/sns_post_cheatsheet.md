# ▼ 共通ルール

## 基本設定

- **投稿時間**：08:00 / 12:00 / 22:00（X・Threads共通・毎日）
- **ハッシュタグ**：なし
- **FW比率（週21本・唯一の正）**：MBTI 7本 ／ ラブタイプ 7本 ／ タイプ言及なし 5本 ／ DSKB 2本（月2本上限。週配分ではなく月内で消化）
  - DSKB月2本を使い切った月は DSKB枠を「タイプ言及なし」に振る（MBTI7／ラブタイプ7／なし7）
  - この数値を他ファイル・他セクションに転記しない。参照する場合は本セクションへのリンクのみにする（数値の二重管理による食い違いを防ぐため）
- **内容の根拠**：`reference/` のみ（架空データ禁止）

## 時間帯と型

| 時間 | 型 | 読者の状態 |
|---|---|---|
| 08:00 | 型B（解釈転換） | 通勤中・視点を変える気づき |
| 12:00 | 型A（シーン共感） | 昼休み・流し見でも止まる |
| 22:00 | 型A または 型C | 就寝前の内省 |

## 4種の型

**型A（シーン共感）**
行動シーンで共感 → 構造解説 → タイプ名に着地 → 他者投影
- 他者投影：「心当たりある人の顔が浮かびませんか？」「これ、周りの誰かに当てはまりませんか？」（明示的なシェア促しは禁止）
- ラブタイプ投稿：キャラクター名で共感 → 行動シーンで展開 → アルファベットで固有化 → 圧縮フレーズ

**型B（解釈転換）**
冒頭は下記4パターンから選択。週3本で同じパターン禁止。
- 観察型：「〇〇な設計がある。」「〇〇する人を見ていると、共通点がある。」
- 問いかけ型：「なぜ〇〇してしまうのか、ずっと気になっていた。」
- 独り言型：「これ、ずっと言いたかった話です。」
- 従来型：「〜という経験がある人へ。」（週1本まで）

**型C（問い＋自答）**
問いを立てて自分で答える。「どう思いますか？」で終わらせない。

**型D（DSKB・エンタメ×分析）**
設計官がDSKBで人間の行動を面白く解析する。月2本。Threadsに向いている。
- 「ドスケベ」文脈には乗らない
- 締め定型：「どれかひとつでも当てはまる人は、設計の話をしましょう。」
- 停止基準：4本連続で保存数が通常投稿の2倍未満 → 一時停止

## ネタローテーション

比率目安：痛み・繰り返し系 全体の過半数以内 ／ 出口・発見系 一定数を必ず含める（生成対象期間の本数に応じて按分）

### 痛み・繰り返し系（最大5本）
| カテゴリ | シーン例 |
|---|---|
| 別れ後 | SNS確認・連絡しようとしてやめる |
| 関係中 | 言いたいことが言えない・尽くしすぎる・合わせすぎる |
| 選択 | 付き合えそうな人を遠ざける・好きじゃない人に安心する |
| 診断後（問題） | 調べたのに変わらない |
| 繰り返し | また同じタイプ・別れた理由が前回と同じ |

### 出口・発見系（最低2本）
| カテゴリ | シーン例 |
|---|---|
| 設計が変わった瞬間 | 「ああ、これか」と気づいた体験 |
| タイプを知る面白さ | 診断名がついた瞬間の「しっくり感」 |
| 相性の化学反応 | 特定タイプの組み合わせ・なぜ惹かれるか |
| 診断後（出口） | 設計図が手に入った感覚 |
| 許可を与える | 「〇〇でいい」型の解放 |
| 感情の分解 | 「〇〇と△△は別の話」 |

使用済みネタ確認 → `posts_x.txt` / `posts_threads.txt` の直近保存分を確認する（別ファイルでの記録管理はしない）

### 採用しない型（高インプレッション・低保存率パターン）

| 型 | 排除理由 |
|---|---|
| 攻撃的ラベリング型（「異常者」等） | 文体設計違反・炎上リスク |
| 議論誘発型（「〇〇って偏見だよね」系） | リプライ多・保存率極低・転換率ゼロ |
| エンタメ消費型（DSKBの笑い文脈） | 保存されず有料記事への動線にならない |

## トーン・文体ルール

| 使う | 禁止 |
|---|---|
| じゃない / じゃなくて | ではない / ではなく |
| けど | でも（接続詞） |
| 〜てしまう | 体言止め連打 |
| 〜んです / 〜なんです（1投稿1回まで） | 〜だ（多用禁止） |

禁止記号：`""` ダブルクォート　／　`──` ダッシュ

- 読点の間引き：「して、やめる」→「してやめる」
- 疑問文は疑問符
- 空行でリズム

中学生上位レベルを基準（平易に・幼稚にしない）。

**「設計」**：バリエーション：`設計が動く` / `設計を変える` / `設計ミス` / `設計図` / `設計がズレた`（禁止：`設計が狂っていた`）　※使用方針は下記「設計ワードの使い方ルール」を参照

**「回路」**：不可抗力感の文脈のみ。1投稿1回まで。それ以外は `パターン` / `癖` / `動き方` に置き換え。

**技術語の連続禁止**：`処理` / `コスト` / `組み込まれている` / `機能していない` は1文に2語以上連続禁止

差し替え辞書：
| 元の表現 | 修正後 |
|---|---|
| コストとして処理する回路が組み込まれている | 自動で避けてしまう癖がある |
| 傷つく前に終わらせる処理が先に走って | 傷つく前に自分で終わらせてしまうパターンがある |

## 逆転・反転構造（週1本以上必須）

高反響パターン。X金言フレーズに最優先で使う。**週1本の必須枠は下記の型α（2行逆転型）で充足する。別途カウントしない。**

| 型 | 例 |
|---|---|
| 「〇〇じゃなくて△△だっただけ」 | 「未練じゃなくて設計に名前がついていない状態だっただけ」 |
| 「〇〇してるんじゃなくて△△してる」 | 「聞いてないんじゃなくて、聞きすぎてる」 |
| 「〇〇したくないんじゃなくて△△したいだけ」 | 「仲良くしたくないんじゃなくて役割として分けたいだけ」 |

**型α：2行逆転型（バズ投稿分析 2026-06-22追加・週1本必須化）**
説明なし・補足なし。2行の対比だけで完結させる。直感に反する命題をA行に、さらに予想を裏切る命題をB行に置く。
- 参考：「理由のある『好き』は、いつか冷める。／『なんか好き』は、一生狂う。」（68.8万インプ・ブクマ1,260）
- MBTICODE変換例：「好きな気持ちに理由をつけようとすると、冷める。／『なんか合う』だけで動いた設計は、一番長く残る。」

シーンの具体度：「連絡しようとしてやめる」→「文章を打って送らずに消す」レベルまで落とす。

## 二つの温度型（バズ投稿分析 2026-06-22追加・週1本試験枠）

相反する感情・スタンスが「矛盾せずに同時に存在できる」構造。読者が「自分の言葉として持ち帰りたい」と感じたときにブクマが起きる。
- 参考：Post 6（ブクマ率0.95%・この分析で最高）
- 恋愛設計変換例：「もうこの人のことを考えなくていい、という気持ちと／でもあの関係が何だったか知りたい、という気持ちは／どちらかを選ばなくていい。両方が同時に本当のことなんです。」
- **注意**：この型の締めに「なんです」を使うと効果的。制限は「〜なんです語尾の制限」セクション参照。

## 発話モードの定義（バズ投稿分析 2026-06-22追加）

X投稿には「解説モード」と「内声モード」の2種がある。現行は解説モードが多すぎる。

| モード | 特徴 | 使う場面 |
|---|---|---|
| 解説モード | 「〜設計があります」「〜の話です」 | 型A・FW説明が必要なとき（週3本まで） |
| 内声モード | 設計官の内声として語る。説明しない | 型α（2行逆転）・型β（二つの温度）・型B独り言型 |

内声モード例：
- NG（解説）：「断れない設計になっているんです。ISFJの話です。」
- OK（内声）：「断れないんじゃない。断るという選択肢が、最後にしか出てこないだけ。」

## ブックマーク誘発設計（金言テスト）

締め前の圧縮フレーズに「この一文だけスクショしたくなるか？」を自問する。
- 合格基準：逆説・対比 ／ 構造に名前をつける ／ 自分への問いになる ／ 抽象的な感覚に固有名詞を与えている
- 不合格例：「設計を変えることで関係の見え方が変わります」（説明的）
- 合格例：「連絡できなかったんじゃなくて、設計が先に手を止めてしまってた。」
- 命名合格例：「情緒のサービス料がゼロな関係」「設計の終着点」「手放せない構造」
- **参照基準**：「燃え尽きたんじゃなくて、断れない設計が蓄積しただけ」をこのアカウントの金言最低基準とする。これ以下の強度は書き直す

## ブックマーク設計の3動機（2026-06-22追加）

ブックマークは以下の3つの動機のどれかが発動したときに起きる。投稿を生成したら必ずどれかに該当するか確認する。

| 動機 | 発動条件 | 投稿の型 |
|---|---|---|
| ①後で読み返したい | AとBの「区別」が示されている。後で確認したくなる | 「AとBは別の話」型 |
| ②誰かに送りたい | 大切な人のことが頭に浮かぶ具体性がある | 他者投影型・リポスト設計 |
| ③自分のものにしたい | この言葉を自分の生き方に組み込みたくなる | 二つの温度型・命名型 |

**「共感で終わる投稿」はブックマークされない。**
共感（そうそう）と発見（え、そういうことだったのか）は別物。発見は「知っていたようで名前がついていなかったものに名前がついた」ときに起きる。

## 「当たり前感」の解消ルール（2026-06-22追加）

「当たり前感」の原因：既知の感情を言語化しているだけで、新しい視点・命名・区別がない。

解消の2パターン：
- **意外な命名**：感情の原因として読者が予想しない言葉を置く。例「選んでいたのはタイプじゃなくて、自分との距離感だった。」
- **AとBの区別**：混同されていた2つを切り分ける。例「『未練』と『終わりの判断材料が揃っていない状態』は別の話。」

## 「なった」「なります」の使い分けルール（2026-06-22追加）

- **「なります」は使わない**：説教・処方箋になる。断言が強すぎて読者の余地を消す
- **「なった」も弱い**：発信者の個人体験で閉じてしまい読者を置いてきぼりにする
- **最適解**：「変化の内容＋なれた/変われた」で書く
  - NG：「その見方に変わってから、少し楽になった。」
  - NG：「その見方に変わると、少し楽になります。」
  - OK：「その見方に変わったとき、責める方向が変われた。」（変化の内容が具体的＋「変われた」で余地を残す）

## 締めパターン多様化ルール（2026-06-22追加）

**「心当たりある人の顔が浮かびませんか？」は1バッチ（9本）で1本まで。**

| 種別 | パターン | 使用頻度 |
|---|---|---|
| 他者投影型 | 「心当たりある人の顔が浮かびませんか？」 | 1バッチ1本まで |
| 問いかけ型 | 「あなたはどちら側にいますか？」「これ、当てはまりますか？」 | 複数可 |
| 独白型 | 「〜は別の話なんです。」（余韻止め） | 複数可 |
| 命名型 | 「それを〇〇と呼ぶことにした。」 | 週1〜2本 |
| 許可型 | 「〜でいい、という話です。」 | 週1〜2本 |
| 沈黙型 | 問いなし・金言1行のみで終わる | 週1本・最高難度 |

## 「設計」ワードの使い方ルール（2026-06-22追加・2026-06-22改訂）

**「設計」は自然に溶け込む場合のみ使う。無理に入れない。**

## 「〜なんです」語尾の制限（2026-06-22追加）

1投稿につき1回まで。2回以上は講義調になりAI感が出る。

## 未解決設計の原則

保存は「また見返したい」ときに起きる。「わかった」で終わると消費される。

- 「なぜそうなるか」か「どうすれば変えられるか」のどちらか一方を意図的に書かない
- 両方書いた投稿は共感で消費されいいねは増えるが保存されない
- 空白にした問いが有料記事への自然な橋渡しになる

## 改善トリガー

| 症状 | 対応 |
|---|---|
| 22時投稿の反応が薄い | 型Cの自答部分を強くする |
| 08時投稿が伸びない | 型Bの解釈転換の切り口を変える |
| Noteフォロワーが増えない | 自己リプライの補足文を「無料で読める」に変更 |
| 保存数が少ない | 金言テストの基準を上げる・締め前フレーズを書き直す |

| インプレッションは高いが転換しない | 採用しない型に近いコンテンツが混入している。ネタローテーションを点検する |

→ 評価語を削除し、**構造の核だけを恋愛パターンへ変換**して使う

## Step 2：変換角度を1行で確定

## Step 4：本文を書く

**書き出し（1〜2行）**

| 使う | 禁止 |

| けど | でも（接続詞） |
| 〜んです / 〜なんです（1投稿1回まで） | 〜だ（多用） |
| 〜てしまう | 体言止め連打 |

- 「設計」は動詞文脈のみ可（「設計が動く」「設計を変える」）。「〜設計があります」はNG

**金言フレーズ（核心1行）**
以下のいずれかを満たす：逆説 ／ 対比 ／ 感情・構造に名前をつける ／ 自分への問いになる

**未解決設計の原則**
「なぜそうなるか」か「どうすれば変えられるか」のどちらか**一方だけ書く**

**タイプ言及なしの場合**

- [ ] 140字以内・ハッシュタグなし

→ 複数案は出力しない。修正依頼があった場合のみ次案を出す。
