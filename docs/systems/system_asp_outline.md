ASPブログ記事のタイトルと見出し構成を設計するシステム。`/asp-outline` で起動。

**Why:** /asp-kaigiで参入判定・/asp-themeでテーマ確定後、記事執筆に入る前にタイトルと構成を確定させるステップが必要。テーマ叩き台をそのままタイトルに使うと最適化されないため専用エンジンを設置。

**How to apply:** ASPブログ記事の執筆前に必ず /asp-outline を通す。ワークフローは `/asp-kaigi` → `/asp-theme` → `/asp-outline` → 執筆の順。

## システム仕様

- コマンドファイル：repo内 `.claude/commands/asp-outline.md`
- 入力：/asp-theme の出力（テーマ・ペルソナ・SEO勝算・記事骨子）
- 複数テーマがある場合は「何番から設計しますか？」と確認してから開始

## 出力内容（テーマごと4項目）

1. **タイトル案3パターン**：ベネフィット直球型・共感状況型・疑問問いかけ型。推奨1案を理由付きで明示
2. **見出し構成（H2/H3）**：【表】【リスト】【レビュー挿入】を付記。4〜6個のH2、各H2に2〜4個のH3
3. **内部リンク設置推奨箇所**：/asp-themeで提案された他テーマへの接続を最低1箇所
4. **上位表示再現チェック**：5項目チェックリスト。✗があれば即修正案を提示

## 設計哲学（4原則）

- 原則1：検索行動の完結（ランディング時に核情報が視認できる）
- 原則2：情報の階層化（概要→詳細→補足）
- 原則3：言語的フック（ベネフィットをタイトル最左部に配置）
- 原則4：比較・照合コストの最小化（3事象以上は必ず表・リスト化）
