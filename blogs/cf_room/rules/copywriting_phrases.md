# cf_room コピーライティング構成ルール（PASONA・唯一の正）

文章パート（フック文・ベネフィットカードの文言・CTA等）を書く際の構成基準。元資料（Desktop上のライティング.txt、約2000フレーズ収録の単語帖）の骨格を要点抽出したもの。以後の更新はこのファイルのみで行う。具体的なフレーズ・文例が必要な場合は `rules/copywriting_phrase_bank.md`（元資料の精製版・2026-07-12統合済み）を参照する。Desktop上の元資料は統合済みのため以後不要。

---

## PASONAの法則（全体構成）

**Problem（問題）→ Affinity（親近感）→ Solution（解決策）→ Offer（提案）**

コピーライティングは「何を、どう言うか」より「どの順番で言うか」の構成力が勝負。特に冒頭3ステップが書き手の腕の見せ所。

- **Step1 問題の明確化**：読み手が緊急に解決すべき「痛み」を明確にする
- **Step2 親近感の醸成**：提案に耳を傾けてもらえるよう、読み手との距離を縮める
- **Step3 解決策の紹介**：商品そのものではなく、解決へのアプローチ（技術・仕組み）を紹介する

### 鍵となる質問（コンセプト設計時に使う）
1. 「この商品を20秒で説明しただけで『売ってくれ』と頼み込んでくる客は、どんな客か？」→ 理想的な読者像を見出す
2. 「なぜこの商品はその悩みを簡単に・短時間で解決できるのか？」「それを聞いた読者はどんな疑いを持つか？」「その疑いを吹き飛ばす具体的な証拠は？」→ 商品の「強み」を掘り当てる

**cf_roomでの実務適用**：Affinityは「筆者が読者と同じ悩みを抱えていた」という個人的な体験談を捏造しない（専業レビュアーとして書いている場合は虚偽になる）。代わりに「読者一般が抱えがちな悩み」＋「筆者の専門性（普段から様々な製品を検証している等、事実に基づく立場）」でAffinity/Authorityを構成する。

## Solution（解決）パートの表現

「重要なポイントを示す」：問題への解決策を一言で言い切れない場合、「重要なポイントがある」とだけ伝えて興味を引き、詳細は後述する構成が有効（例：「◯つのポイント」「◯へのカギ」）。

## Offer（提案）パートの表現

- **独自性・優位性（USP）の強調**：「なぜ他からではなく、あなたから買うのか？」に答える。価格だけの差別化は消耗戦になるため、価格以外の独自性を打ち出す
- **商品価値を高める5つの技術**：
  1. 異なるカテゴリーの商品と比較する（例：甘酒＝「飲む点滴」）
  2. 複数の価格帯を用意する（スタンダード/デラックス等）
  3. 特典を用意する
  4. 保証をつける（満足度保証・成果保証）
  5. 商品名を直感的にわかりやすくする（専門用語だけの命名は避ける）
- **「解決策」と「提案」の違い**：解決策は一般的な手段（例：ダイエット）、提案は今回読み手にすすめる具体的な手法・条件。誇大表現（120%に見せる）はNG、魅力を100%伝えることに留める

## フレーズカテゴリ一覧（用途別・目次的に使う）

見出しやCTA、ベネフィットカードの短文を書く際、以下のカテゴリから状況に合う言い回しの方向性を選ぶ。具体例が必要な場合は `rules/copywriting_phrase_bank.md` を参照。

| カテゴリ | 用途 |
|---|---|
| 欲求・欲望に訴える | モテる・稼ぐ・儲かる等、根源的欲求に直結させたいとき |
| 質問を投げかける | 読み進めさせたい・行動を促したいとき |
| 好奇心をそそる | 「理由」「なぜ」「こうして」等、続きが気になる形にしたいとき |
| ギャップを生む | ビフォーアフター・意外性を強調したいとき |
| 比較で興味を引く | A vs B、できる人／できない人などの対比 |
| 注意を促し注目を集める | 「やってはいけない」「警告」等、離脱抑止したいとき |
| Affinity（親近感） | 読み手との距離を縮めたいとき（実体験のみ使用、捏造禁止） |
| ストーリー性を出す | 一発逆転・体験談を語りたいとき |
| 誘う・仲間意識を強める | コミュニティ感・共感を出したいとき |
| 誠実さ・親切さを出す | 信頼構築を重視したいとき |
| 方法を提示する | ◯つの方法・使い方・活用法を伝えたいとき |
| 簡単さを強調する | 誰でも・やさしい・ラクして等、ハードルの低さを伝えたいとき |
| 効率性にフォーカス | たった◯分で・時短・コスパ等 |
| 期待を高める | いま・ついに・とうとう等、待望感を出したいとき |
| 秘密の雰囲気を醸し出す | 「秘密」「秘訣」「誰も教えてくれない」等 |
| 学びの要素を強調する | 教科書・レッスン・◯◯に学ぶ等 |
| 新登場・新常識訴求 | 新発売・新食感・非常識な・革命等、刷新感を出したいとき |

## cf_room固有の適用ルール

- Affinity（親近感）は必ず事実ベースで書く。個人の実体験の捏造は絶対禁止（専業レビュアーであれば「様々な製品を検証している立場」として書く）
- 見出し・CTAの言い回しに迷ったら、上記カテゴリ一覧から目的に合うものを選び、`headline_writing.md`のケープルズ4原則（得・新情報・好奇心・簡単さ＋具体的数字）と掛け合わせて作る
- 「解決策」の説明と「提案（Offer）」の説明を混同しない。まず一般的な仕組みを解説し、その後cf_room記事としての具体的なおすすめ・結論を分けて書く
