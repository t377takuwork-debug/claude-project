## 1. H2タイムテーブル直下のリードパラグラフを必ず更新する

`カウントダウンTV 最新タイムテーブル` H2見出しの直下に、以下の形式のリードパラグラフが存在する。

```html
<p>2026年MM月DD日（曜）19:00から放送の『CDTVライブ！ライブ！』出演アーティストと披露曲です。<br>※現在は公式発表前の構成予測です。</p>
```

このパラグラフは `cdtv-rewrite.md` の Grepマーカー一覧に掲載されていないため、バッチ更新時に見落としやすい。

**Why:** Batch B でタイムテーブル本体を更新してもこのリードパラグラフが前回の日付のまま残り、ユーザーが手動修正することになった（6/29→7/6の更新で発生）。

**How to apply:** Batch B に独立した Edit として必ず含める。old_string は `から放送の『CDTVライブ！ライブ！』` の前後の日付部分で特定する。

## 2. 締め文は2段落構成にする

`[nopc][mokujimae][/nopc]` の直後に置く締め文は、1段落の長い文ではなく **2段落に分割する**スタイルが好まれる。

```html
<!-- wp:paragraph -->
<p>[nopc][mokujimae][/nopc]</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>【核心情報1文】</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>【リアルタイム更新告知1文】</p>
<!-- /wp:paragraph -->
```

**構成パターン例（7/6版）:**
- 1段落目：「7月6日のCDTVはYOASOBIが2週連続プラチナライブ！」（今回の核心）
- 2段落目：「その他アーティストのテレビ初披露曲など、オンエアに合わせて出演順をリアルタイム更新していきます！」（更新告知）

**Why:** 核心情報と更新告知を同じ段落に詰め込むと冗長になる。分割することでスマートフォンでの読みやすさも向上する。

**How to apply:** Batch B の締め文 Edit では、最初から2段落構成で出力する。「今回はXXXが〇〇！〇〇など、リアルタイムで更新していきます！」のような1文連結は避ける。

## 3. 「放送時間傾向パラグラフ」も放送時間に合わせて更新する

`[nopc][originalsc][/nopc]` の直前に、放送構成の傾向を説明するパラグラフが存在する。

```html
<p>2時間生放送では前半（19時台）→後半（20時台）の2段構成で、後半ほど注目アーティストが配置される傾向です。</p>
```

3時間SPから通常2時間回（またはその逆）に切り替わる際に、ここが前回の放送時間のまま残りやすい。

**Why:** Grepマーカー外のセクションで、Batchから漏れやすい。7/6リライト時に「3時間生放送では〜」が残ったままになっていた（ユーザーの確認指摘で発覚）。

**How to apply:** Batch B に含める。2時間通常回は `2時間生放送では前半（19時台）→後半（20時台）の2段構成`、3時間SPは `3時間生放送では前半→中盤→後半で役割が分かれやすく` と書き分ける。
