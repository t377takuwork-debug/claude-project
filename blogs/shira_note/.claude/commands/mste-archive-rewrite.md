# ミュージックステーション（Mステ）月次アーカイブ記事リライト

Mステの月次アーカイブ記事を指定月の内容にリライトし、`draft_mste_archive.txt` として上書き保存する。

`/cdtv-archive-rewrite` と同一の運用方針・作業原則を踏襲する（Editベース・トークン効率重視）。Mステは**週次放送**（通常1時間枠、月内に3〜5回）のため、CDTVの「月2回」前提の箇所は放送回数に応じて調整する。

## 事前情報の確認

リライト開始前に以下の情報が揃っているか確認する。不足している場合はユーザーに確認する。

### 必須情報（資料①）
- 対象月（YYYY年MM月）
- 当月の放送回数と各放送日（例：7/4、7/11、7/18、7/25）
- 各放送回ごとの情報：
  - 出演アーティスト一覧（出演順）
  - 各アーティストの歌唱曲
  - 特記事項（初披露・フルサイズ・◯周年・大トリ・企画コーナーなど）
  - 推定タイムライン（時刻・アーティスト・曲名）
  - 通常回（1時間）か2時間SPかの区分
- 記事URL（PAGE_URL：`https://shira-treat.com/〇〇/`）
- 記事の初回公開日（DATE_PUBLISHED）

### 任意情報
- アイキャッチ画像URL（変更がある場合のみ）

### 確認すべき曖昧点はAskUserQuestionで一括確認する

以下は毎回発生しやすい曖昧点。個別に何度も聞き返さず、該当するものをまとめて1回のAskUserQuestion呼び出しで確認する（2026-07-14実績：3点をまとめて1回で確認し完了）。

| 曖昧点 | 確認内容 |
|---|---|
| 出演順データの食い違い | `draft_mste.txt`（速報記事）に既存の出演順・熱量分析がある放送日について、資料①で別の分析（自由形式のSNS再現分析等）が渡された場合、両者を比較し新旧どちらを採用するか確認する。差分（アーティストの順番・★ピークの対象）を具体的に提示すること |
| 当月の放送回数 | 資料①で言及された放送日が当月の全放送回か、他にも放送済み回がある可能性がないか確認する（Mステは月2回程度の実績が多いが月3〜5回の場合もある） |
| DATE_PUBLISHED | 前月と異なるPAGE_URLの場合（月ごとに新規記事URLを使う運用のため）、その記事の初回公開日時を確認する |

## 作業方針（トークン効率重視・Editベース）

| 原則 | 内容 |
|---|---|
| **ショートコードは独立ブロック** | `[nopc][title]`・`[nopc][mokujimae]`・`[nopc][originalsc]` はそれぞれ独立した `<!-- wp:paragraph -->` ブロックに分割する |
| **Editで差分更新が基本・Writeは使わない** | 2ヶ月目以降は前月分がすでに正しい構造を持つため、タイトル・メタ・aside・スケジュール表・まとめ1文・JSON-LD可変欄など「変わる箇所」だけをEditで置換する。日付別セクションは内容が全面的に変わるため、そのブロック単位でEdit置換する。Write（全体上書き）は、draft自体が存在しない・破損している場合の初回作成時のみ使用する |
| **傾向・見逃し配信ブロックは一切触れない** | 手順6の2ブロック（出演順・タイムテーブルの傾向／見逃し配信・再放送）は月をまたいで完全固定。Read・Edit・出力のいずれも行わない（前月の内容がそのまま残る） |
| **まとめブロックは1文字列のみEdit** | 手順6の3ブロック目は「YYYY年M月」を含む1文だけをEditで差し替える。関連リンク・アフィリエイトHTMLには触れない |
| **文字数カウントはPowerShellを使う** | `$s.Length` で確認する。Bashツールは出力が返らない既知の問題があるため文字数カウント用途では使わない |
| **ユーザー修正後の確認** | 修正後はsystem-reminderの差分を優先参照し、JSON-LD整合性など必要箇所のみRead・確認する |

## WordPressブロックコメント規則（必須）

ドラフトファイル全体を通じて、以下のブロックコメントを必ず使用する。

| コンテンツ種別 | 形式 |
|---|---|
| テキスト段落 | `<!-- wp:paragraph --><p>本文</p><!-- /wp:paragraph -->` |
| H2見出し | `<!-- wp:heading --><h2>見出し</h2><!-- /wp:heading -->` |
| H3見出し | `<!-- wp:heading {"level":3} --><h3>見出し</h3><!-- /wp:heading -->` |
| HTML全ブロック（aside・table・div等） | `<!-- wp:html -->...<!-- /wp:html -->` |
| 画像 | `<!-- wp:image {"align":"center"} -->...<!-- /wp:image -->` |
| JSON-LD | `<!-- wp:html --><!-- MANUAL_JSONLD_ARCHIVE --><script>...</script><!-- /wp:html -->` |
| **ショートコード（独立必須）** | `<!-- wp:paragraph --><p>[nopc][title][/nopc]</p><!-- /wp:paragraph -->` |

**モデルファイル**: `draft_mste_archive.txt` が正規の出力形式（2ヶ月目以降）。形式に迷った場合はこのファイルを参照する。

**配色**：Mステは`draft_mste.txt`と同じ白背景・ネイビー（#003366）＋ピンク（#ff0084）＋イエロー（#ffdc00）＋ブルー（#00d4ff）のグラデーション基調（CDTVの黒背景ネオン配色とは別デザイン）。

### 2. タイトル・メタディスクリプション

**タイトル**
```
【Mステ】YYYY年M月の出演者や曲目・セトリ一覧（放送アーカイブ）
```
- `【Mステ】` を冒頭に固定
- 年・月を正確に記載（例：2026年7月）
- `（放送アーカイブ）` を末尾に固定

**メタディスクリプション（120字以内）**

ドラフトファイルのタイトル行の直下に以下の形式で記載する。

```
メタディスクリプション（XXX字）：
YYYY年M月放送「ミュージックステーション」のタイムテーブル、出演者、歌唱曲などを過去のアーカイブとして掲載。振り返り用として{月内の最大見どころ1〜2点}もまとめています。
```

#### 作成ルール

| 項目 | 内容 |
|---|---|
| 冒頭 | 「YYYY年M月放送「ミュージックステーション」のタイムテーブル、出演者、歌唱曲などを過去のアーカイブとして掲載。」で固定 |
| 見どころ | 30年ぶり出演・大型コラボ・周年企画など、その月固有の見どころを簡潔に |
| 末尾 | 「まとめています。」で固定 |
| 文字数 | 100〜115字を目安。120字を超えない |

### 4. H2：YYYY年M月のミュージックステーション放送スケジュール

**放送スケジュール表の各行を当月の放送日で更新する。**

| 要素 | 更新内容 |

| 放送日・曜日 | 正確な日付・曜日に更新 |

### 5. H2：M月DD日（曜日）のタイムテーブル・セットリスト（新しい日付から降順）

**放送日ごとにH2セクションを1つ作成する。新しい放送日が上に来るよう降順で並べる。**

#### 5-a. 出演者・歌唱曲一覧（`draft_mste.txt`のARTIST LINEUPカードグリッドを踏襲・2026年7月版で確定）

**このカードグリッド形式が唯一の正。** 2026年6月以前のアーカイブ（`draft_mste_archive.txt`の古いセクション）はテーブル形式（`<table>`）だったが、2026年7月版でカードグリッドに統一した。今後迷う必要はない。

| カード要素 | 更新内容 |
|---|---|
| セクション見出し | 「ARTIST LINEUP (YYYY.MM.DD)」 |
| カード1枚 = 1組 | アーティスト名（太字）＋「〇〇を披露。」の1文 |
| 特記アーティスト（初出演・◯周年・大トリ等） | ピンク系背景（`#fff8f8`〜`#fff0f6`・border `#ffcce0`）＋バッジで強調 |
| 通常アーティスト | 白〜グレーグラデーション背景（`#f8fafc`〜`#f1f5f9`・border `#dce4ed`） |

放送前・出演順未確定の回は「出演順は放送後確定」と注記し、判明している出演者を順不同で列挙する（5-bのタイムラインパネルは省略）。

#### 5-b. タイムライン（実際の出演順アーカイブ）

`draft_mste.txt`の「実際の出演順【放送後更新】」アーカイブパネル（`mst-archive-panel`・グラデーション枠・タイムライン形式）を1日分ずつ踏襲する。

| タイムライン要素 | 更新内容 |
|---|---|
| タイトルバー | 「M月DD日 熱量・セットリストレポート」 |
| MAIN CAST | MC・主な出演者を記載 |
| 時刻バッジ | 「HH:MM頃」＋ブロック名（OPENING／EARLY／MID BLOCK／LATE BLOCK等） |
| ★投稿爆発ピーク | その回で最もSNS反響が大きかった出演を強調（ピンク文字・大きめフォント） |

タイムラインの直後に、その放送回の見どころ・SNS反響・特筆すべきパフォーマンスを2〜4文で記載する（`[nopc][originalsc][/nopc]` は使用しない）。

### 7. 構造化データ（JSON-LD）の更新

**前月ドラフトのJSON-LD（手順1でGrepした行範囲）を骨格として使い、下表の可変フィールドだけをEditで差し替える。** `template_jsonld_mste_archive.txt` は、フィールド名や構造を忘れた場合の参照用としてのみ使う（通常は前月ドラフトを見れば足りる）。

| プレースホルダー | 差し替え内容 |
|---|---|

| `HEADLINE` | タイトル（WordPressタイトル欄と同じ文字列） |
| `META_DESCRIPTION` | メタディスクリプションと同じ文字列 |

| `EYECATCH_URL` | アイキャッチ画像URL |
| `MONTH_JA` | 対象月（例：2026年7月） |

| 項目 | 内容 |
