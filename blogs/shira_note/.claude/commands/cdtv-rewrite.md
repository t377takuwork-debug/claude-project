## 作業方針

| 原則 | 内容 |
|---|---|
| **WriteしないでEditする** | 変更箇所のみEditで差分更新。Write（全体上書き）は使用しない |
| **静的セクションはスキップ** | 見逃し配信・ナビゲーション/アフィリエイトブロックはRead・Edit不要（ドラフトの既存ブロックをそのまま維持する。`template_cdtv_static.txt` はバックアップ参照用） |

| **ユーザー修正後の確認** | 修正後はsystem-reminderの差分を優先参照し、JSON-LD FAQ整合性など必要箇所のみRead・確認する |

### 1. ファイル準備（前回ドラフト全体を読まない）

以下の順序でファイルを読み込む。

**① テンプレートを Read する**
- `template_cdtv_static.txt`：スキップ可。見逃し配信・ナビゲーションブロックはドラフトの既存ブロックを維持する（手順9・13参照）。内容を確認・復元したい場合のみ Read する
- `template_jsonld_cdtv.txt`：JSON-LD のプレースホルダー構造を確認

**② 直前回ドラフトのアーカイブセクションのみ Grep → Read で抽出する**
```
対象ファイル: C:\Users\PC_User\claude project\blogs\shira_note\draft_cdtv.txt
Grep パターン: "カウントダウンTV 過去のタイムテーブル一覧"
  → ヒット行番号を確認し、Read offset=<その行> limit=160 で抽出
```
取得目的：H3フルカード2枚・details 内データ・アーカイブリンクを次回ドラフトに反映するため。

**③ 更新完了後、並行Editバッチでセクションを差分更新する**

全体Writeは使用しない。以下のバッチ単位で1メッセージに複数Editを束ねて送信する。

| バッチ | 含めるEdit |
|---|---|
| **Batch A** | タイトル・メタ ／ aside BROADCAST日付 ／ aside LINEUP ／ aside 注目バナー |
| **Batch B** | 締め文 ／ H2タイムテーブル見出し（日付） ／ タイムテーブル本体（丸ごと） ／ originalsc注記 |
| **Batch C** | 出演者一覧H2（日付） ／ 出演者一覧リード文 ／ 出演者カードグリッド全体 ／ まとめリード文 |
| **Batch D** | 出演時間カード（目安・オープニング・特別企画） ／ 順番の傾向（FLOW以外の4枚） ／ LIVE日付 ／ まとめパラグラフ |
| **Batch E** | 実際の出演順（テーブル全体 ＋ 冒頭注記 ＋ 末尾注記） |
| **Batch F** | アーカイブセクション全体（H3フルカード2枚 ＋ details折りたたみ） |
| **Batch G** | FAQ（Q1〜Q4） ／ まとめH2本文 ／ JSON-LD全体 |

**冒頭段落（固定文）はEditしない。** `[nopc][title][/nopc]` はすでに独立ブロックになっているため、手を加えない。

### 2. タイトル・メタディスクリプション

**タイトル**
```
【CDTV】タイムテーブル M月DD日速報！出演順・出演時間をリアルタイム更新
```
- `【CDTV】` を冒頭に固定
- 放送日を「M月DD日」形式で記載
- `速報！` を維持してリアルタイム感を出す

**メタディスクリプション（120字以内）**

ドラフトファイルのタイトル行の直下に以下の形式で記載する。

```
メタディスクリプション（XXX字）：
CDTVライブ！ライブ！M月DD日タイムテーブルを速報更新！{最大注目ポイント1}・{注目ポイント2}。出演順・登場時間の目安をリアルタイムで随時追記。
```

#### 作成ルール

| 項目 | 内容 |
|---|---|
| 冒頭 | 「CDTVライブ！ライブ！M月DD日タイムテーブルを速報更新！」で固定 |
| 注目ポイント | マンスリーライブ出演者・コラボ・初出演など、この回固有の見どころを「・」区切りで列挙 |
| 末尾 | 「出演順・登場時間の目安をリアルタイムで随時追記。」で固定 |
| 文字数 | 100〜115字を目安。120字を超えない |

**【SEO】** 「マンスリーライブ」「初出演」「コラボ」など今回固有の言葉を使い、前回と同じ文章にならないようにする。
