## 作業方針（トークン効率重視・Editベース）

| 原則 | 内容 |
|---|---|
| **ショートコードは独立ブロック** | `[nopc][title]`・`[nopc][mokujimae]`・`[nopc][originalsc]` はそれぞれ独立した `<!-- wp:paragraph -->` ブロックに分割する |
| **Editで差分更新が基本・Writeは使わない** | `draft_cdtv_archive.txt` は前月分がすでに正しい構造を持つ。タイトル・メタ・aside・スケジュール表・まとめ1文・JSON-LD可変欄など「変わる箇所」だけをEditで置換する。日付別セクション（手順5）は内容が全面的に変わるため、そのブロック単位でEdit置換する（旧セクション全体を新セクション全体に一括置換）。Write（全体上書き）は、draft自体が存在しない・破損している場合の初回作成時のみ使用する |
| **傾向・見逃し配信ブロックは一切触れない** | 手順6の2ブロック（出演順・タイムテーブルの傾向／見逃し配信・再放送）は月をまたいで完全固定。Read・Edit・出力のいずれも行わない（前月の内容がそのまま残る） |
| **まとめブロックは1文字列のみEdit** | 手順6の3ブロック目は「YYYY年M月」を含む1文だけをEditで差し替える。関連リンク・アフィリエイトHTMLには触れない |
| **文字数カウントはPowerShellを使う** | `$s.Length` で確認する。Bashツールは出力が返らない既知の問題があるため文字数カウント用途では使わない |
| **ユーザー修正後の確認** | 修正後はsystem-reminderの差分を優先参照し、JSON-LD整合性など必要箇所のみRead・確認する |

### 2. タイトル・メタディスクリプション

**タイトル**

**放送スケジュール表の各行を当月の放送日で更新する。**

| 要素 | 更新内容 |

### 5. H2：M月DD日（曜日）のタイムテーブル・セットリスト（新しい日付から降順）

**放送日ごとにH2セクションを1つ作成する。新しい放送日が上に来るよう降順で並べる。**

#### 5-a. 歌唱曲一覧（放送状況で形式を使い分ける）

**【放送済み回】テーブル形式**

| カード要素 | 更新内容 |
|---|---|
| ヘッダー | 「M月DD日 出演アーティスト・歌唱曲一覧」 |
| ARTIST列 | 出演者名（出演順に並べる） |
| SONG列 | 歌唱曲名。複数曲は ` ／ ` 区切り |
| バッジ（ピンク） | フルサイズ・初披露・TV初披露など特記事項 |
| バッジ（黄色） | マンスリーライブ週数・特設ブロック |
| バッジ（グレー） | 衣装・演出などの補足 |

テーブルの行は偶数・奇数で背景色を交互にする（`#000` / `#050505`）。

**【放送前・事前発表のみの回】カードグリッド形式**

2カラムカードグリッド（通常アーティスト）＋特設ハイライトカード（マンスリーライブ・周年企画・特別フェス等）を使用する。

- **通常カード**: `background:#111; border:1px solid #222;` + 上部グラデーションライン（`#ff008d → #ff00ea`）。アーティスト名（白・太字イタリック）と楽曲名（黄色・10px）を記載
- **ハイライトカード**: `border:1px solid #ff008d;`（マンスリー/周年）または `border:1px solid #222;` + 黄色グラデーション（特別フェス）。バッジ・楽曲一覧・説明文付き
- 出演順未確定の場合は「出演順は放送後確定」と注記し、50音順またはセグメント別に整理する

`draft_cdtv_archive.txt` の最新月の放送前セクションが実装例。

#### 5-b. タイムライン（出演順アーカイブ）

タイトルバーの `M月DD日 SETLIST TIMELINE` を更新する。

| タイムライン要素 | 更新内容 |
|---|---|
| 時刻バッジ | 「HH:MM頃〜」（19時台は白背景・黒文字、20時台は黄色背景・黒文字） |
| アーティスト名 | 通常は白文字。特設ブロック・マンスリー枠はピンク文字 |
| 曲名 | グレー文字で小さく表示。複数曲は改行 |
| 特設ブロック | ピンク時刻バッジ・ピンクアーティスト名・詳細説明付きで強調 |

タイムラインの直後に、その放送回の見どころ・SNS反響・特筆すべきパフォーマンスを2〜4文で記載する（`[nopc][originalsc][/nopc]` は使用しない）。

### 7. 構造化データ（JSON-LD）の更新

**前月ドラフトのJSON-LD（手順1でGrepした行範囲）を骨格として使い、下表の可変フィールドだけをEditで差し替える。** `template_jsonld_cdtv_archive.txt` は、フィールド名や構造を忘れた場合の参照用としてのみ使う（通常は前月ドラフトを見れば足りる）。

#### 差し替え箇所

| プレースホルダー | 差し替え内容 |
|---|---|
| `PAGE_URL` | 記事URL（末尾スラッシュ付き） |
| `HEADLINE` | タイトル（WordPressタイトル欄と同じ文字列） |
| `META_DESCRIPTION` | メタディスクリプションと同じ文字列 |
| `DATE_PUBLISHED` | 記事初回公開日時（ISO形式：YYYY-MM-DDTHH:MM:SS+09:00） |
| `DATE_MODIFIED` | 作成・更新日時（ISO形式：YYYY-MM-DDTHH:MM:SS+09:00） |
| `EYECATCH_URL` | アイキャッチ画像URL |
| `MONTH_JA` | 対象月（例：2026年5月） |
| `ARTIST_KEYWORDS` | 月内全出演者名をカンマ区切りで列挙（`"名前", "名前"` 形式） |
| `N_BROADCASTS` | 当月の放送回数（整数） |
| `BREADCRUMB_NAME` | パンくずリスト最終階層テキスト（例：2026年5月アーカイブ） |
| `BROADCAST_EVENT_LIST` | 各放送日のListItem（ひな形を放送回数分展開、新しい日がposition:1） |

#### ItemListの各BroadcastEventに記載する内容

| 項目 | 内容 |
|---|---|
| `name` | 「CDTVライブ！ライブ！（YYYY年MM月DD日放送）」 |
| `startDate` | 放送開始ISO形式（通常：`YYYY-MM-DDT19:00:00+09:00`） |
| `endDate` | 放送終了ISO形式（通常枠：`YYYY-MM-DDT21:00:00+09:00`、3時間SP：`22:00` など） |
| `performer` | 当回出演者をMusicGroup/Person型で列挙 |

#### アーティストの型判定

| 区分 | 型 |
|---|---|
| グループ・バンド | `{ "@type": "MusicGroup", "name": "アーティスト名" }` |
| ソロ（個人名） | `{ "@type": "Person", "name": "アーティスト名" }` |

#### 固定箇所（変更不要）

- `author` / `publisher`
- `about`（CDTVの番組情報）
- `BreadcrumbList` の1・2階層目
