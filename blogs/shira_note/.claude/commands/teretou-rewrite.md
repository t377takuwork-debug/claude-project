## 作業方針（必読）

| 原則 | 内容 |
|---|---|
| **WriteしないでEditする** | 変更箇所のみEditで差分更新。Write（全体上書き）は使用しない |
| **静的セクションはスキップ** | ナビゲーション/アフィリエイトブロック（まとめH2直下の `<!-- wp:html -->` div）はRead・Edit不要 |
| **ショートコードは独立ブロック** | `[nopc][title]`・`[nopc][mokujimae]`・`[nopc][originalsc]` はそれぞれ独立した `<!-- wp:paragraph -->` ブロックに配置する。テキストと同一ブロックに混在させない |
| **全体Readしない** | Grepで対象セクションの行番号を特定してからoffset/limitで必要範囲だけ読む |
| **独立セクションは並行Edit** | old_stringが互いに重複しない場合は1メッセージに複数Editを束ねる |

### WPブロック形式のEditルール

| 要素 | Edit時のold_string/new_string の範囲 |
|---|---|
| 段落テキスト | `<!-- wp:paragraph -->\n<p>` から `</p>\n<!-- /wp:paragraph -->` まで含める |
| H2見出し | `<!-- wp:heading -->\n<h2>` から `</h2>\n<!-- /wp:heading -->` まで含める |
| HTMLブロック内の特定要素 | `<!-- wp:html -->` は含めない。変更対象の具体的な内容文字列のみを old_string に指定する |
| タイトル・メタ（先頭行） | プレーンテキストをそのまま対象にする（WPブロックなし） |

### Grepフェーズ（作業開始時に全マーカーを1メッセージで並行実行）

## リライト手順

### 1. タイトル・メタディスクリプション

ファイル先頭のプレーンテキスト行を直接Editする（WPブロックなし）。

**タイトル**
```
【YYYY夏】テレ東音楽祭タイムテーブル｜出演者の順番や出演時間まとめ(随時更新)
```
- 年号（YYYY）を更新。「テレ東音楽祭」冒頭・季節（夏）を固定。

**メタディスクリプション（120字以内）**

| 項目 | 内容 |
|---|---|
| 冒頭 | 「テレ東音楽祭YYYY夏」出演者・タイムテーブルをどこよりも早く随時更新！（年号を更新） |
| 注目ポイント | 今年の目玉（初出演アーティスト・記念ステージ等）を優先 |
| 放送日 | MM/DD（曜）HH:MM放送 を含める |
| 文字数 | 90〜115字を目安。120字を超えない |

---

### 2. 冒頭文（イントロ段落）

`<!-- wp:paragraph -->` ブロックごとEditする。

| 段落 | 更新内容 |
|---|---|
| 1段落目 | 回数・放送日・曜日・時刻を更新（`今年でXX回目を迎える〜YYYY年MM月DD日（曜）夜HH:MMから放送決定！`） |
| 2段落目 | 今年固有の特記事項（初の〇〇放送・放送時間変更等）を更新 |

---

### 3. asideカード（プレビュー）

`<!-- wp:html -->` ブロック内の以下の要素を個別にEditする。

| 更新要素 | Editの対象文字列（old_string の核心） |
|---|---|
| ヘッダーの年号・季節 | `MUSIC FESTIVAL</span> <span ...>2026 SUMMER</span>` の `2026 SUMMER` 部分 |
| ON AIR 日付・時間 | `>2026年6月28日（日）夜6:30～ 生放送<` の日時部分 |
| MC行 | `>MC：深澤辰哉・松本若菜 / 進行：田中瞳<` のMC名部分 |
| 注目企画バナー | 見どころ `<span>` テキストを今年の目玉企画に書き換え（毎年変わるためReadで確認後Edit） |

---

### 4. 締め文（`[nopc][mokujimae][/nopc]` 直後の段落）

今年の見どころ・放送日を1〜2文で書き換える。WPブロックごとEditする。

今年固有の企画名（ZARDの復活ステージ→今年の目玉）を具体的に盛り込む。

### 5. H2：タイムテーブルと出演時間

**年が変わる場合のみ** H2見出し内の年号（YYYY）を更新する（WPブロックごとEdit）。

H2直下のリード文（`<!-- wp:paragraph -->`）を出演組数・発表状況に合わせて書き換える。

**タイムテーブルasideカードの更新**

| 要素 | タイムテーブル未発表時 | タイムテーブル発表後 |
|---|---|---|
| ヘッダー年号 | `MUSIC FES 2026` の年号を更新 | 同左 |
| 各時間帯の見出し | 現状維持 | 確定した時間帯ラベルに更新 |
| 各時間帯説明文 | 今年の出演者名を根拠にした具体的な予測文に更新 | 確定した出演者・楽曲・企画内容に更新 |
| 注記テキスト | 今年の放送日・発表状況に合わせて更新 | 「公式確定」に変更 |

時間帯説明文は「今年の出演者名を根拠にした具体的な内容」として書く。汎用文（「豪華アーティストが登場」等）は使わない。

### 8. H2：発表スケジュール予測

**出演者追加のみの資料①ではスキップ可。** 発表タイミングの新たな確定・実績追加がある場合のみ更新する。

| 要素 | 更新内容 |

### 12. 構造化データ（JSON-LD）の更新

JSON-LDはファイル末尾の `<!-- MANUAL_JSONLD -->` で始まる `<!-- wp:html -->` ブロック内。更新フィールドの値はcontextから直接構築できるため、**ReadなしでEditを実行してよい**（Grepで行番号を得たらそのままEditへ進む）。

#### 毎回更新する箇所

| フィールド | 更新内容 |
|---|---|
| `description` | メタディスクリプションと同じ文字列 |
| `dateModified` | 更新日（ISO形式：YYYY-MM-DDTHH:MM:SS+09:00） |
| `keywords` | 今年の出演者名・企画等を更新 |
| BroadcastEvent `name` | `テレ東音楽祭YYYY夏` の年号・季節を更新 |
| BroadcastEvent `description` | 今年の主な見どころを含む放送説明文 |
| BroadcastEvent `startDate` / `endDate` | 放送日時を更新（`YYYY-MM-DDTHH:MM:SS+09:00`） |
| FAQPage `mainEntity` | 記事本文のQ&Aと**一字一句一致**させる（3問） |

#### 変更がある場合のみ更新する箇所

| フィールド | 更新タイミング |
|---|---|
| `headline` | タイトルを変更した場合 |
| `image` | 新しいアイキャッチ画像URLが提供された場合 |

#### 固定箇所（変更不要）

- `datePublished` / `dateCreated`：2026-06-03T00:00:00+09:00（固定）
- `@id` / `url`（記事URL）
- `author` / `publisher`
- `about`（テレ東音楽祭番組情報）
