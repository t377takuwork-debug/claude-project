# Mステ タイムテーブル記事リライト

Mステ（ミュージックステーション）のタイムテーブル記事を最新情報にリライトし、`draft_mste.txt` として上書き保存する。

## 事前情報の確認

リライト開始前に以下の情報が揃っているか確認する。不足している場合は以下の順序で対応する。

1. **WebSearch で最新情報を検索する**（例：「Mステ YYYY年MM月DD日 出演者」）
2. 取得できた場合は「検索で以下の情報が取得できました。内容を確認してください」とユーザーに提示し、確認を取ってからリライトを進める
3. 検索でも情報が見当たらない場合のみ「情報が見当たりません。資料①をご提供ください」と返す

**注意：検索せずに「未発表」「情報なし」と断言しない。**

### 必須情報（資料①）
- 放送日（YYYY年MM月DD日）
- 放送時間・チャンネル
- 出演アーティスト一覧
- 各アーティストの披露曲
- 今回の見どころ・注目ポイント（コラボ・初披露・特別企画など）
- VTR企画の有無と内容

### 推奨情報（資料②）
- 直前回のX（旧Twitter）投稿分析データ
  - 各アーティストの出演時間帯・順番
  - SNS反応のピーク・盛り上がりポイント
  - 「調査結果」として過去一覧カードに反映する

### 2. タイトル・メタディスクリプション

**タイトル**
```
Mステ タイムテーブル【M月DD日】出演順・出演時間をリアルタイム更新
```
- 「Mステ タイムテーブル」を冒頭に固定
- 放送日を【】で表記

**メタディスクリプション（120字以内）**

ドラフトファイルのタイトル行の直下に以下の形式で記載する。

```
メタディスクリプション（XXX字）：
Mステ（M月DD日）タイムテーブルを速報更新！{最大注目ポイント1}・{注目ポイント2}・{コラボや企画}。出演順・登場時間の目安を随時追記。
```

#### 作成ルール

| 項目 | 内容 |
|---|---|
| 冒頭 | 「Mステ（M月DD日）タイムテーブルを速報更新！」で固定（鮮度シグナル） |
| 注目ポイント | 初披露・ラスト出演・コラボなど、この回固有の見どころを「・」区切りで列挙 |
| 末尾 | 「出演順・登場時間の目安を随時追記。」で固定（検索意図との一致） |
| 文字数 | 100〜115字を目安。120字を超えない |
| アーティスト名 | 最大注目の2〜3組を優先して入れる。全員を列挙しない |

### 3. 冒頭段落・aside カード

以下を更新する。

| プレースホルダー | 内容 |
|---|---|
| 放送日時 | YYYY年MM月DD日(曜) HH:MM〜HH:MM |
| STATUS | 「出演アーティスト解禁」など現時点の状態 |
| ARTISTS | 出演者を `/` 区切りで列挙 |
| TIMETABLE（見どころ） | 今回の最大注目ポイントを1〜2文で |
| 編集者コメント | 初披露・ラスト出演など特記事項 |
| 最終更新日 | 記事作成日（YYYY.MM.DD） |

締め文（`[nopc][mokujimae][/nopc]` 直後の段落）は今回の見どころに合わせて自然な文に更新する。

> **ショートコードのブロック分割ルール**：`[nopc][title]`・`[nopc][mokujimae]`・`[nopc][originalsc]` はそれぞれ独立した `<!-- wp:paragraph -->` ブロックに配置する。前後のテキストと同一ブロックに混在させない。

**【SEO】** 締め文は「推しの出番を逃したくない」という検索者の切実な動機（ペルソナ：佐藤真由）に直接応えるコピーにする。一般論ではなく「今回だからこそ見る理由」を1文で示す。

### 4. H2：最新タイムテーブル
カード内のヘッダーの日付・出演者バッジを更新する。
予測の時間帯・注目ポイントの説明文は今回の情報に合わせて書き換える。

### 5. H2：出演者・歌唱曲一覧
各アーティストカードを最新の出演者・楽曲に差し替える。
`ARTIST LINEUP (YYYY.MM.DD)` の日付も更新する。

### 6. H2：出演時間は何時頃？【目安】
3ブロック（序盤・中盤・終盤）の説明を今回の出演者・見どころに合わせて書き換える。
カードヘッダーの日付（例：`6/12 出演時間帯の予測ガイド`）も更新する。

**【SEO】** 各ブロックの説明文は「過去の傾向＋今回の出演者を組み合わせた独自予測」として書く。「一般的に序盤は〜」といった汎用文ではなく、今回の顔ぶれを根拠にした具体的な予測にすることで一次情報としての価値を出す。

### 7. H2：Mステの順番の傾向
カード内の「今週の注目パフォーマンス」「大トリの予測」「VTR企画のタイミング」を今回の情報で更新する。
通常回/特番の構成説明は共通のため原則維持。

**【SEO】** 「今週の注目パフォーマンス」「大トリの予測」は断言調で書く（「〜の可能性があります」より「〜が最有力」）。検索者は答えを求めており、曖昧な表現は離脱につながる。根拠（過去の傾向・今回の特殊事情）を短く添えることで信頼性を補う。

### 8. H2：実際の出演順【放送後更新】
**放送前は直前回（資料②）のアーカイブデータを暫定掲載する。**
アーカイブパネル内の日付・出演者・タイムライン・REPORTを直前回の情報に差し替える。
放送後にリアルタイムで今回の実データに書き換える運用。

### 9. H2：見逃し配信・再放送はある？

**`template_mste_static.txt` の「BLOCK: 見逃し配信セクション」をそのままコピーする。変更不要。**

YouTube埋め込みURLが古い場合のみ差し替える（現在：`https://www.youtube.com/watch?v=dlA_57GHVaI`）。

### 10. H2：過去のタイムテーブル一覧
以下のルールで更新する。

| 操作 | 内容 |
|---|---|
| 追加 | 直前回（資料②のデータ）をフルカード形式でH3として先頭に追加 |
| 維持 | 2回前の回はH3フルカードのまま維持 |
| 移動 | 3回前の回を `<details>` 折りたたみブロックの先頭に移動（コンパクト形式に変換） |
| 削除 | `<details>` 内で最も古い回を削除 |

フルカード形式（H3直下のwp:html）→ コンパクト形式（details内のシンプルなリスト）への変換は自動で行う。

### 11. H2：よくある質問（FAQ）
Q1〜Q4の質問・回答を今回の出演者・見どころに合わせて書き換える。
- Q1: タイムテーブルの事前発表（定型文）
- Q2: 今回最大の注目アーティストの出番予測
- Q3: 特別企画・コラボの時間帯予測
- Q4: 見逃し配信の有無（定型文）

**【SEO】** Q文は「読者が検索バーに実際に打ち込む言葉」で書く。「〜はありますか？」「〜は何時頃？」など口語的な疑問形が検索意図と一致しやすい。A文は冒頭の1文で結論を出し、補足説明を続ける構造（結論→根拠）にする。

### 12. H2：【まとめ】
放送日・出演者・見どころを今回の情報で書き換える。

### 13. ナビゲーション・アフィリエイトブロック（固定）

**`template_mste_static.txt` の「BLOCK: ナビゲーション／アフィリエイトブロック」を JSON-LD の直前にそのままコピーする。変更不要。**

### 14. 構造化データ（JSON-LD）の更新

ドラフトファイル末尾のJSON-LDを以下のルールで更新する。

#### 差し替え箇所

| 項目 | 更新内容 |
|---|---|
| `headline` | タイトル（WordPressタイトル欄と同じ文字列） |
| `description` | メタディスクリプションと同じ文字列 |
| `dateModified` | 記事作成・更新日時（ISO形式：YYYY-MM-DDTHH:MM:SS+09:00） |
| `image` / `thumbnailUrl` | 今回のアイキャッチ画像URLに更新 |
| `mentions` / `performer` | 出演者をMusicGroup/Person型のJSON配列で列挙 |
| BroadcastEvent `name` | 「ミュージックステーション（YYYY年MM月DD日放送）」に更新 |
| BroadcastEvent `description` | 今回の放送説明文に更新 |
| BroadcastEvent `startDate` / `endDate` | 放送日時（ISO形式）に更新 |
| ItemList `name` | 「YYYY年MM月DD日 Mステ出演者と歌唱曲一覧」に更新 |
| ItemList `numberOfItems` | 出演者数に更新 |
| ItemList `itemListElement` | 出演者を最新に更新 |
| FAQPage `mainEntity` | 記事本文のQ&Aと**一字一句一致**させる（3〜4問） |

#### 固定箇所（変更不要）

- `datePublished` / `dateCreated`：2025-11-17T00:00:00+09:00（固定）
- `author` / `publisher`
- `about`（Mステ番組情報）
- `BreadcrumbList`

#### アーティストの型判定

| 区分 | 型 |
|---|---|
| グループ・バンド | `{ "@type": "MusicGroup", "name": "アーティスト名" }` |
| ソロ（個人名） | `{ "@type": "Person", "name": "アーティスト名" }` |

#### 注意事項

- アイキャッチ画像URLはリライト前にユーザーに確認する
- FAQのQ&A文は記事本文と**一字一句一致**させる（Googleの整合性チェック対策）
