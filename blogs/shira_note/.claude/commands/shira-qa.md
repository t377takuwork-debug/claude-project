# /shira-qa — ドラフト検品コマンド

リライト完了後のドラフトを機械検品し、検出された問題を修正するまでを1コマンドで行う。

## 引数

- `/shira-qa draft_musicday.txt` — 指定ファイルを検品
- `/shira-qa all` または引数なし — 全draftを検品

## 手順

### ① 検品スクリプトを実行する

```powershell
cd "C:\Users\PC_User\claude project\blogs\shira_note"
python tools\qa_draft.py {対象ファイル名}        # 全件なら --all
```

スマートクォートが検出された場合のみ、`--fix` を付けて再実行してよい（自動修正されるのはスマートクォート置換のみで、他の内容は変更されない）。

### ② 結果の解釈ルール

| 区分 | 対応 |
|---|---|
| ERROR | **必ず修正する**。修正後に再度スクリプトを実行し、ERROR 0件を確認する |
| WARN（禁止ワード） | 本文・カード説明文・締め文なら修正する。**アーティスト名・曲名・番組公式の企画名に含まれる場合は修正しない**（例:「彩」を含む人名） |
| WARN（メタ申告字数不一致） | `メタディスクリプション（XX字）：` の XX を実字数に直す |
| WARN（ul style / カテゴリURL） | 該当箇所を修正する（ul→div変換ルールは下記参照） |
| WARN（リード文の日付不一致） | 最新回の放送日と照合し、古ければ更新。**放送日が最頻日付と異なるだけの場合（例: 未来の放送日）は問題なし**とユーザーに報告 |
| INFO（日付分布） | 修正不要。分布に違和感があればユーザーに報告のみ |

### ③ 修正時の必須ルール

- 修正は Edit ツールで行い、**修正後に必ず ① を再実行**して ERROR 0件・対象WARN解消を確認する
- ul→div 変換: `<ul style=...>` → `<div style=...>`（list-style系CSSは削除）、`<li>` → `<div>`（シンプル箇条書きは先頭に `・` を追加）
- ショートコード分離: `[nopc][xxx][/nopc]` は必ず単独の `<!-- wp:paragraph --><p>...</p><!-- /wp:paragraph -->` に分離する
- メタディスクリプション修正時は JSON-LD の `description` も一字一句同じ文に更新する

### ④ 完了報告

以下の形式で報告する:

```
## 検品結果: {ファイル名}
- ERROR: {N}件 → {修正した/なし}
- WARN: {N}件 → {修正 M件 / 意図的に残置 K件（理由）}
- 最終ステータス: ✅ ERROR 0件
```

## 運用ルール

- 各番組のリライトコマンド（/mste-rewrite 等）の最終ステップとして必ず本コマンド相当の検品を実行する
- 検品スクリプトが検出できない項目（文体・事実関係・出演者網羅）は従来どおり各リライトコマンドのチェックリストに従う
