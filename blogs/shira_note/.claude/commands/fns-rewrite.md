## 作業方針

- **差分EditのみでWriteしない**
- **全体Readしない**：Grepで行番号取得 → offset/limitで最小範囲Read → Edit
- **並行Edit**：old_stringが重複しない手順は1メッセージに束ねる
- **ショートコード独立ブロック**：`[nopc][title]`・`[nopc][mokujimae]`・`[nopc][originalsc]` は単独 `<!-- wp:paragraph -->` に配置
- **禁止ワード**：memory `feedback_shira_rewrite_banned_words` 参照。「彩る」「飾る」「幕を開ける」「見せ場」「筆頭に」「フィナーレを」「締めくくります」等は使用しない

### WPブロック Edit の範囲

| 要素 | old_string の範囲 |
|---|---|
| 段落テキスト | `<!-- wp:paragraph -->\n<p>` ～ `</p>\n<!-- /wp:paragraph -->` |
| H2見出し | `<!-- wp:heading -->\n<h2` ～ `</h2>\n<!-- /wp:heading -->` |
| HTMLブロック内要素 | `<!-- wp:html -->` は含めない。変更対象の文字列のみを指定 |
| タイトル・メタ（先頭行） | プレーンテキストをそのまま対象にする（WPブロックなし） |

> **注意**：メタディスクリプションとJSON-LDの `description` は同一文字列になる。メタ行をEditするときは `メタディスクリプション（XX字）：\n` を含む文脈を old_string に含めて重複マッチを防ぐ。

## リライト手順

### 1. タイトル・メタ

先頭のプレーンテキスト行をEditする（WPブロックなし）。

| 要素 | 内容 |
|---|---|
| タイトル | `FNS歌謡祭 タイムテーブル＆出演順番は？YYYY年夏の最新情報` |
| メタ | 90〜115字（120字以内）。放送日・目玉企画・行動喚起を含める |

### 2. 冒頭文

| 段落 | 内容 |
|---|---|
| 1段落目 | 放送日・曜日・時刻・番組名を更新 |
| 2段落目 | 今年固有の見どころ（コラボ・企画名）を事実で並べる |

### 3. プレビューカード

| 更新要素 | 内容 |
|---|---|
| 年号バッジ | `<span ...>YYYY</span>` の年号 |
| ON AIR 日時 | `YYYY年MM月DD日（曜）HH:MM〜HH:MM 生放送` |
| PICK UP COLLAB | 今年の注目コラボ・企画名 |
| 下部リード文 | 今年の企画・特色を反映した1文 |

### 4. 締め文（`[nopc][mokujimae][/nopc]` 直後の段落）

**1文のみ。** タイムテーブル未発表時は「現時点では未発表。当日ブックマーク推奨」の旨で今年の企画名を含める。

### 5. H2：タイムテーブルと出演順番

年号変更時のみH2見出しを更新。H2直下リード文を出演組数・発表状況に合わせて書き換える。

**タイムテーブルカード：**

| 要素 | 未発表時 | 発表後 |
|---|---|---|
| 時間帯見出し | 現状維持 | 確定ラベルに更新 |
| 時間帯説明文 | 今年の出演者名を根拠にした予測文（汎用文NG） | 確定出演者・楽曲・企画内容 |
| 出演者リスト | 公式発表ベースの予測リスト | 確定リスト |

### 6. H2：出演者・グリッドカード・PROJECTSセクション

年号変更時のみH2見出しを更新。H2直下リード文を今年の出演総数・注目点で書き換える。

**グリッドカード：**

| 操作 | 内容 |
|---|---|
| 年号バッジ | `YYYY SUMMER` |
| 各 `artist-card` | アイウエオ順。初登場は `pink-border` クラス＋`color: #ff007f;` |
| 楽曲確定 | 楽曲名を記載 |
| 楽曲未発表 | `順次発表` |
| コラボ | `コラボ` |

**PROJECTSセクション：**

| カード | 内容 |
|---|---|
| 豪華コラボ | 今年のコラボ企画（説明文1文） |
| 注目初登場 | 今年の初登場アーティスト名 |
| 映画・周年 | 今年の記念企画（なければ別テーマ可） |

### 7. H2：見どころカード（`fns-project-section`）

H2直後段落を今年版に更新。横スクロールカード（4〜5枚）を今年の企画で書き換える。

| 要素 | 内容 |
|---|---|
| `strong` | 今年の企画名 |
| `p` | 出演者名・楽曲・見どころを1〜2文 |
| 枠線カラー | コラボ=#ff007f・K-POP=#00b4d8・周年=#f59e0b・初登場=#0d9488 |

### 8. H2：発表スケジュール（新情報がある場合のみ）

過去実績を残しつつ今年の予測を更新する。

| 要素 | 内容 |
|---|---|
| 第1弾発表行 | 確定日・MC・組数を更新。過去実績を1行追加 |
| 追加発表行 | 発表済み弾数・傾向を更新 |
| タイムテーブル行 | 直近年実績＋今年の予測を追加 |
| CHECK!ボックス | 公式X・チェック時期を更新 |

### 9. H2：FAQ

| 問 | 更新内容 |
|---|---|
| Q1 | 放送日時・曜日・放送時間 |
| Q2 | 今年のタイムテーブル解禁タイミング予測 |
| Q3 | 配信情報（TVerは放送当日に公式確認した実態を反映） |

### 10. H2：まとめ

| 要素 | 内容 |
|---|---|
| 冒頭段落 | 放送日・今年固有の企画名 |
| 「注目の出演アーティスト」カード | 総組数・注目アーティスト名 |
| 「タイムテーブル・順番の公開時期」カード | 今年の解禁時期予測 |
| 「最新情報をチェック」カード | 当サイトの更新案内（変更不要なら維持） |
| ブックマーク促進段落 | 放送日・放送時間 |

---

### 11. JSON-LD

ファイル末尾（`MANUAL_JSONLD` マーカー直後）。ReadなしでGrepの行番号からEditへ進む。

#### 毎回更新

| フィールド | 内容 |
|---|---|
| `description` | メタと**一字一句同じ**文字列 |
| `dateModified` | 更新日時（YYYY-MM-DDTHH:MM:SS+09:00） |
| `keywords` | 今年の出演者名・企画等 |
| BroadcastEvent `name` | `YYYY FNS歌謡祭 夏` |
| BroadcastEvent `description` | MC・出演総組数・見どころ |
| `startDate` / `endDate` | 放送日時 |
| FAQPage `mainEntity` | 記事本文Q&Aと**一字一句一致** |

#### 変更時のみ更新

| フィールド | タイミング |
|---|---|
| `headline` | タイトル変更時 |
| `image` | 新アイキャッチURL提供時 |

#### 固定（変更不要）

`datePublished`（初公開日）・`@id` / `url`・`author`・`publisher`・`about`

## 表現多様化・視聴メモ（全番組共通・2026-07-16追加）

- 文言を生成・更新する全手順で `rewrite_common_rules.md` **10章（表現多様化・視聴メモ・予測の書き方）** を適用する
- 更新予告・未発表案内・CTA等の機能フレーズは、前回リライトと同じ言い回しを使い回さない（10章のバリエーション表からローテーション。同一文の記事内3回以上リピートはqa_draftがWARN検知）
- **資料③（視聴メモ）が渡された回のみ**、10章の仕様（`class="shicho-memo"`）で視聴メモブロックを挿入する。渡されない回は挿入せず、AIが感想を創作しない
- 独自予測はクッション言葉でデータと主観を区別する（10-3）
- 完了報告に「この回の独自要素」を1行含める（10-4 HCSセルフチェック）
