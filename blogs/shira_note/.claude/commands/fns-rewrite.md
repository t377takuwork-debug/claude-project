## 作業方針（必読）

| 原則 | 内容 |
|---|---|
| **WriteしないでEditする** | 変更箇所のみEditで差分更新。Write（全体上書き）は使用しない |
| **静的セクションはスキップ** | まとめH2直下の他番組ナビゲーション・Amazon/楽天アフィリエイトリンクはRead・Edit不要 |
| **ショートコードは独立ブロック** | `[nopc][title]`・`[nopc][mokujimae]`・`[nopc][originalsc]` はそれぞれ独立した `<!-- wp:paragraph -->` ブロックに配置する。テキストと同一ブロックに混在させない |
| **全体Readしない** | Grepで対象セクションの行番号を特定してからoffset/limitで必要範囲だけ読む |
| **独立セクションは並行Edit** | old_stringが互いに重複しない場合は1メッセージに複数Editを束ねる |

### WPブロック形式のEditルール

| 要素 | Edit時のold_string/new_string の範囲 |

## リライト手順

### 1. タイトル・メタディスクリプション

ファイル先頭のプレーンテキスト行を直接Editする（WPブロックなし）。

**タイトル**
```
FNS歌謡祭 タイムテーブル＆出演順番は？YYYY年夏の最新情報
```
- 年号（YYYY）を更新。「FNS歌謡祭」「夏の最新情報」を固定。

**メタディスクリプション（120字以内）**

| 項目 | 内容 |
|---|---|
| 冒頭 | 放送日・放送時間を明記（例：「YYYY年7月DD日生放送の『FNS歌謡祭 夏』」） |
| 注目ポイント | 今年の目玉企画・コラボ・初登場アーティストを優先 |
| 行動喚起 | タイムテーブル確認・ブックマーク促進 |
| 文字数 | 90〜115字を目安。120字を超えない |

---

### 2. 冒頭文（イントロ段落）

`<!-- wp:paragraph -->` ブロックごとEditする。

| 段落 | 更新内容 |
|---|---|
| 1段落目 | 放送日・曜日・時刻・番組名を更新（`YYYY年MM月DD日（曜）HH:MMから放送されるフジテレビ系の夏の大型音楽特番『YYYY FNS歌謡祭 夏』`） |
| 2段落目 | 今年固有の見どころ（コラボ・企画の予告など）に更新 |

---

### 3. asideカード（プレビュー）

`<!-- wp:html -->` ブロック内の以下の要素を個別にEditする。

| 更新要素 | Editの対象文字列（old_string の核心） |
|---|---|
| ヘッダーの年号 | `FNS歌謡祭 夏 <span ...>2026</span>` の年号部分 |
| ON AIR 日付・時間 | `2026年7月1日（水）18:30〜21:54 生放送` の日時部分 |
| PICK UP COLLABテキスト | 今年の注目コラボ・企画出演者名に書き換え |
| 下部リード文 | 今年の企画・特色を反映した一文に書き換え |

### 4. 締め文（`[nopc][mokujimae][/nopc]` 直後の段落）

今年の見どころ・放送日を1〜2文で書き換える。WPブロックごとEditする。

タイムテーブル未発表の場合は「現時点では未発表。当日ブックマーク推奨」の旨を維持し、具体的な企画名を今年版に更新する。

### 5. H2：タイムテーブルと出演順番

**年が変わる場合のみ** H2見出し内の年号（YYYY）を更新する（WPブロックごとEdit）。

H2直下のリード文（`<!-- wp:paragraph -->`）を出演組数・発表状況に合わせて書き換える。

**タイムテーブルカードの更新**

| 要素 | タイムテーブル未発表時 | タイムテーブル発表後 |
|---|---|---|
| 各時間帯の見出し | 現状維持（`18時台`・`19時台`・`20時台`・`21時台`） | 確定した時間帯ラベルに更新 |
| 各時間帯説明文（`<p>`） | 今年の出演者名を根拠にした具体的な予測文（汎用文NG） | 確定した出演者・楽曲・企画内容に更新 |
| 各時間帯の出演者リスト | 今年の公式発表を反映した予測リスト | 確定出演者・楽曲名を反映 |

時間帯説明文（`<p style="...">`）は今年の出演者名を明示した具体的な内容とする。汎用文（「豪華アーティストが登場する」等）は使わない。

### 11. 構造化データ（JSON-LD）の更新

JSON-LDはファイル末尾の `<!-- wp:html -->` ブロック内（`<!-- MANUAL_JSONLD -->` が直後の行に続く構造）。更新フィールドの値はcontextから直接構築できるため、**ReadなしでEditを実行してよい**（Grepで行番号を得たらそのままEditへ進む）。

#### 毎回更新する箇所

| フィールド | 更新内容 |
|---|---|
| `description` | メタディスクリプションと同じ文字列 |
| `dateModified` | 更新日（ISO形式：YYYY-MM-DDTHH:MM:SS+09:00） |
| `keywords` | 今年の出演者名・企画等を更新 |
| BroadcastEvent `name` | `YYYY FNS歌謡祭 夏` の年号を更新 |
| BroadcastEvent `description` | 今年のMC・出演総組数・見どころを含む放送説明文 |
| BroadcastEvent `startDate` / `endDate` | 放送日時を更新（`YYYY-MM-DDTHH:MM:SS+09:00`） |
| FAQPage `mainEntity` | 記事本文のQ&Aと**一字一句一致**させる（3問） |

#### 変更がある場合のみ更新する箇所

| フィールド | 更新タイミング |
|---|---|
| `headline` | タイトルを変更した場合 |
| `image` | 新しいアイキャッチ画像URLが提供された場合 |

#### 固定箇所（変更不要）

- `datePublished` / `dateCreated`：2026-06-03T00:00:00+09:00（固定）
- `@id` / `url`（記事URL）：https://shira-treat.com/fnskayousai-timetable/
- `author` / `publisher`
- `about`（FNS歌謡祭番組情報）
