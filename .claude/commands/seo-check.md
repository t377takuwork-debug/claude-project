# /seo-check — 記事URL×キーワード SEO診断コマンド

任意の記事URLとターゲットキーワードを渡すと、機械チェック（`blogs/seo/content_seo_checker.py`）とClaudeによる質的判断を組み合わせ、SEO診断レポートを1本にまとめる。

## 引数

- `/seo-check <URL> <キーワード1>[,キーワード2,...] [--compare]`
- URLまたはキーワードが渡されていない場合は「診断対象のURLとターゲットキーワードを教えてください」と一言聞いてから開始する。
- `--compare` を付けた場合のみ手順④（競合比較）を実行する。付けない場合は機械チェック＋Claude判定のみ（従来どおり）で完了する。

## 手順

### ① 機械チェックを実行する

```powershell
python "C:\Users\PC_User\claude project\blogs\seo\content_seo_checker.py" {URL} {キーワード（カンマ区切り。1つ目=メイン）}
```

出力される `完了。レポート: {パス}` の行からレポートパスを控える。実行エラー（URL取得失敗等）はそのままユーザーに報告し、以降の手順は行わない。

### ② レポートと記事本文を読む

- ①で出力されたMarkdownレポートをReadで読む
- ③の質的判断のため、対象URLをWebFetchで取得し実際の記事本文を読む。取得・抽出に失敗した場合は「本文取得に失敗したため質的判断は保留」と明記し、③をスキップして④以降に進む

### ③ 「要確認(人的判断)」項目にClaudeの見解を追記する

レポート内「SEO_guide.txt 判断チェックリスト対照」表のうち、機械判定できず `要確認(人的判断)` になっている「設計」「執筆」フェーズについて、実際に読んだ記事本文を根拠に判定する。

| フェーズ | 確認ポイント | 判定基準 |
|---|---|---|
| 設計 | 「最後に知りたいこと」が記事の起点（冒頭）にあるか | 冒頭2〜3段落に結論・核心情報があるか。前置きや一般論から入っている場合は⚠️ |
| 執筆 | 独自視点・一次情報が含まれているか | 実体験・独自データ・具体的記述があるか。誰が書いても同じ一般論の焼き直しなら⚠️ |

**判定は必ず記事本文からの具体的な引用・要約を根拠として示す。根拠を示せない場合は判定せず「要確認(人的判断)」のまま残す（捏造しない）。**

**AI特有の当たり障りのない平均的な評価を避けるための基準（重要）：**
- 「読者目線でわかりやすい」「情報が整理されている」など、**どの記事にでも言えてしまう定型的な誉め言葉だけ**で判定しない。この記事固有の具体的な記述（数値・固有名詞・独自の言い回し）を根拠にする
- それらしいセクション（傾向分析・考察等）が存在する＝自動的に✅ではない。実際に独自の解釈・法則性を導出しているか、単なる事実の並べ替え・公式発表の言い換えに過ぎないかを見極める
- 性善説で採点しない。根拠が薄い・当てはまるか微妙な場合は✅にせず「要確認(人的判断)」のまま残すか⚠️にする
- 全項目が✅で終わる場合でも、改善余地が本当にないか一度疑う。指摘できる弱点があれば「改善アドバイス」に反映する（無理に弱点を作る必要はないが、安易に「問題なし」で済ませない）

Editツールでレポートファイルの該当行を更新する：
- 判定列 → `✅ (Claude判定)` または `⚠️ (Claude判定)`
- 詳細列 → 根拠（記事本文からの具体的な引用・要約、1〜2文）

### ④ 競合比較（`--compare` 指定時のみ）

**目的はあくまで「記事の質としての比較」。被リンク数・ドメインパワー・CTR等、本文からわからない外部要因は評価しない（評価できないため断定しない）。実際の検索順位を予測するものでもない。**

1. WebSearchで「{メインキーワード} {サブキーワード}」（スペース区切りの複合クエリ。サブキーワードがなければメインのみ）を検索し、対象記事自身のドメインを除いた上位3〜5件のURLを控える
2. 各URLをWebFetchし、扱っているトピック・見出し構成・鮮度（更新日の言及）・独自情報の有無を把握する。取得失敗したURLはスキップし、成功した件数を明記する
3. 自記事と競合を次の3軸で比較する。**「競合も同程度」で済ませず、具体的にどのトピックが競合にあって自記事にないか（逆に自記事にしかないか）を最低1つ挙げる。差が本当にない場合のみ「差なし」と明記する**

| 軸 | 比較観点 | 判定 |
|---|---|---|
| トピック網羅度 | 競合が扱っているが自記事にない話題があるか | ✅(同等以上)/⚠️(不足) |
| 鮮度 | 更新頻度・最新情報の反映度が競合と比べてどうか | ✅/⚠️/ℹ️(比較不可) |
| 独自性・一次情報 | 競合も同じ一般論の焼き直しか、どちらかに独自データ・独自視点があるか | ✅/⚠️ |

レポートに「## 競合比較（コンテンツの質のみ／被リンク等は対象外）」セクションをEditツールで追記する。比較した競合URL（何件成功したか）・軸ごとの判定と根拠に加え、**必ず「総括」の1〜2文を含める**：3軸の結果を踏まえて「競合と比べて内容面で見劣りしないか／劣るか」を明言する（被リンク・ドメインパワー等の外部要因は対象外である旨もここで一言添える）。総括を省略しない。

### ⑤ 改善アドバイスをマージする

③（と実行時は④）の判定結果を踏まえ、レポート内「改善アドバイス」セクションに必要な項目のみ箇条書きを追記する（機械チェック側のアドバイスと重複させない）。追記不要なら何もしない。

### ⑥ 完了報告

チャットには以下の形式で提示する：

```
## SEO診断: {URL}
- キーワード: メイン=「X」 / サブ=「Y」
- キーワード含有: {タイトル/メタ/H1/H2H3/本文の要点1行}
- Claude判定: 設計={✅/⚠️/保留} / 執筆={✅/⚠️/保留}（各1行で根拠を添える。定型的な誉め言葉ではなく記事固有の根拠を書く）
- 競合比較（--compare指定時のみ）: 比較件数・軸ごとの判定を各1行
- 「上位表示できる質か」への回答（--compare指定時のみ・必須）: 競合と比べて内容面で見劣りしないか／劣るかを1〜2行で明言する。被リンク・ドメインパワー等は対象外である旨を一言添える
- 改善アドバイス（上位2〜3件。なければ「特になし」と明記）
```

レポートは `blogs/seo/output/latest_report.md` に上書き保存済み（履歴は残さない）。パスは毎回同じなので、完了報告で改めて明示する必要はない。

## 完了条件

レポートファイルに「機械チェック＋Claude判定（根拠つき）」（`--compare`指定時は競合比較も）が保存されており、チャットに要約を提示済みであること。

## 注意

- スコア化・合計点は出さない（`SEO_guide.txt` の方針。`content_seo_checker.py` 側の設計と同一の考え方）
- 記事本文を実際に読まずに「設計」「執筆」フェーズを判定しない
- WebFetch失敗時は判定を保留し、理由を明記する（無理に判定を作らない）
- 競合比較は「記事の質」のみを対象とし、実際の検索順位・被リンク・ドメインパワーには言及しない（言及できるだけの情報がないため）
