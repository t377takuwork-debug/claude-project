# /post-review — SNS投稿 壁打ち・冷徹デバッガー

生成済みのSNS投稿案（X / Threads / リプライ / 引用RT）を投稿前に検品する。
**投稿案を生成するコマンドではない**。生成→本コマンドで壁打ち→修正→提案、の順番を守る（feedback_s4lv_x_post_workflow）。

## 引数

`/post-review {アカウント} {媒体}` 例: `/post-review s4lv X` / `/post-review mbticode reply`
引数がなければ、直前に生成した投稿群を対象とし、アカウント・媒体を文脈から判定する。

## レビューの原則

- **冷徹に採点する。忖度しない。** 各投稿に100点満点のスコアと、減点理由を必ず付ける
- 90点未満の投稿は修正案を併記する
- チェックは「事実 → 構造 → 文体 → 投稿群全体」の順で行う

---

## STEP 1: 事実チェック（全アカウント共通・最優先）

- [ ] 比較データのない成約率・効果を断言していないか
- [ ] 検証前の情報を確定実績として語っていないか（s4lv: Claude Code関連は「テスト運用中」枠のみ）
- [ ] 事実の誇張がないか（「弱」を削る等）
- [ ] 実体験にない内容を語っていないか（該当アカウントの personal_data.md と照合）

**1つでも該当したら即修正。ここで落ちた投稿は文体が良くても出さない。**

## STEP 2: 構造チェック

### s4lv（X投稿）
- [ ] 狙う反響（ブックマーク / いいね・RP / リプライ）が投稿ごとに1つに決まっているか（混在は禁止）
- [ ] 狙いに対応する確定構造か
  - ブックマーク: 情緒→論理（リスト）→情緒のサンドイッチ
  - いいね・RP: エグい比喩→記憶のゆらぎ（身体感覚）→静かな真理
  - リプライ: 弱さの開示→唯一の矜持→属性を絞った問いかけ（余白を意図的に残す）
- [ ] テスト投稿は140字以内。ただし削りすぎ禁止（余裕があるのに情緒要素を削って体温が消えていないか）

### s4lv（Threads投稿）
- [ ] X投稿OSをそのまま持ち込んでいないか（空気感が硬くないか）
- [ ] 文間の1行空け・独白の隙間があるか
- [ ] 語尾が親密距離か（「〜なかったんです。」「〜なんですよね。」）
- [ ] 実績が「愛着」として語られているか（盾・自慢はNG）
- [ ] 弱さの10%トッピングがあるか
- [ ] 絵文字は😅✋のみ・最大2個・150〜250字

### MBTICODE（リプライ・引用RT）
- [ ] 相手への共感・尊重が先にあるか（冒頭の軟らかい否定「〜はわかるんですけど」禁止）
- [ ] INTP-Aの実体験として成立しているか（「わかります」は本当に共感できる体験のみ）
- [ ] 元の投稿にない視点を1つ加えているか
- [ ] 保存したくなるフレーズが1つあるか（対比・命名・逆転・問い）
- [ ] 「設計」ワードを使っていないか / 恋愛を結論・締めに使っていないか

## STEP 3: 文体チェック

### 共通禁止
- [ ] 命令口調（〜しろ / 〜すべき）
- [ ] 短文羅列のAI感（「〜があった。〜なかった。やめた。変わった。」）
- [ ] AI的な因果文（原因→結果が綺麗にまとまりすぎ。感情の反応語で置き換える）
- [ ] 問いかけの口語体（「何だと思う？」→「何だと思いますか？」）

### s4lv X
- [ ] フックは1行か / 強い逆説フックになっていないか
- [ ] 手順は「1. 2. 3.」数字リスト（【Step1】形式NG）・括弧内補足は削る・リストにインデント
- [ ] です/ます と だ/である の意図的混在が保たれているか（統一しない）
- [ ] ライブ感要素（喋り言葉の接続詞 / 記憶のゆらぎ / 思考プロセス10%）があるか
- [ ] 締めは断言が基本（ベネフィットまで見せる）

### MBTICODE
- [ ] 禁止語尾がないか（「〜だよな」/ て形終わり「〜て。」/「〜思って。」多用 /「これ、身に覚えがあって。」）

## STEP 4: 投稿群全体チェック（複数投稿の場合）

- [ ] 170万PVの言及は投稿群全体で1回以内か（s4lv）
- [ ] 信頼性ワードが分散しているか（残りはブログ歴・全削除・100件成約等の別軸で担保）
- [ ] Note誘導は1日1本以内か / URLは本文でなくリプライ欄か
- [ ] 「また別の機会に。」等の次回誘導は1日1回以内か
- [ ] 3本が同じトーン・フックになっていないか

## 出力フォーマット

```
## post-review 結果（{アカウント} / {媒体}）

### 投稿1: {冒頭15字}…
- スコア: {N}/100
- 狙い: {ブックマーク/いいね・RP/リプライ}
- ✅ 通過: {通過項目の要約}
- ❌ 減点: {項目 → 理由}
- 修正案: {90点未満の場合のみ全文提示}

### 投稿群全体
- 170万PV言及: {N}回 / Note誘導: {N}本 / トーン被り: {あり・なし}

### 総合判定: {そのまま投稿可 / 修正後に再レビュー / 再生成推奨}
```
