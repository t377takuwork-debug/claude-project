# /kpi-weekly — MBTICODE 週次KPI記録・前週比分析

ユーザーが渡す数値（`brands/mbticode/kpi_weekly_template.md` 形式・またはスクリーンショット/雑多なテキスト）を受け取り、記録ファイルへの追記と前週比コメントまでを1回で完了する。

## 手順

1. **入力を受け取る**: テンプレ形式でなくてもよい。渡された数値をテンプレの項目（フォロワー3媒体・Note記事・X投稿・Threads投稿・特記事項）に整理する。**渡されていない数値を推測で埋めない**（不明欄は「未取得」と書く）
2. **前週データを読む**: `brands/mbticode/kpi_log.md`（週次スナップショットの蓄積ファイル。初回実行時に新規作成する）の最新エントリを読む
3. **今週分を追記する**: `kpi_log.md` の末尾に「## 集計期間: YYYY-MM-DD〜MM-DD」セクションとして整理済みデータを追記・保存する
4. **前週比コメントを出す**（チャット出力・3点のみ）:
   - 伸びたもの1つ（数値の差分つき）
   - 落ちた・停滞したもの1つ（数値の差分つき）
   - 来週の再判断に関わる事実1つ（例: 分岐判断日 2026-07-19 の条件「引用ポスト平均インプ50超」に対する現在地）
5. **戦略判断はしない**: 「投稿本数を変えるべき」等の方針変更提案はこのスキルの範囲外。判断が必要な兆候があれば「/notekaigi で議論する価値があります」と1行添えるだけに留める

## 完了条件

`brands/mbticode/kpi_log.md` に今週分が追記保存されている＋前週比3点コメントを提示済み。

## 注意

- 数値の捏造・補間は絶対禁止。ユーザーが渡した数字だけを記録する
- 分岐判断の基準値（インプ50超・フォロワー+10〜20人等）の唯一の正は `brands/mbticode/sns_post_cheatsheet.md`「次回判断日」セクション
