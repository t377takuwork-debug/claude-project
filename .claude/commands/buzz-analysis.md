# /buzz-analysis — バズ投稿分析エンジン

ユーザーが手動収集したX/Threads投稿（自分の高反響投稿・他者のバズ投稿）を、固定の観点で分析し、アカウント資産（cheatsheet・見本バンク）への反映案まで出す。

> 前提: 収集は必ずユーザーが行う（スクリーンショットまたはテキスト貼り付け）。X/Threadsのブラウザ自動取得は技術的制限により禁止（`brands/mbticode/rules/project_buzz_analysis.md`）。取得の自動化を提案しない。

## 入力（ユーザーから受け取るもの）

1. 投稿本文（必須）
2. 数値: インプ・いいね・リポスト・リプライ・ブックマーク（わかる範囲）
3. 投稿主の属性: 自分／他者（他者ならフォロワー規模）

数値が一切ない場合は「反響の根拠がないため構造分析のみになります」と明示して進める。

## 分析フレームワーク（1投稿ごとに必ず全観点を埋める）

| 観点 | 判定内容 |
|---|---|
| ① 型 | 型α（2行逆転）／二つの温度型／型A〜D／リスト型／該当なし（`brands/mbticode/sns_post_cheatsheet.md` の型定義に照合） |
| ② 発話モード | 解説モード／内声モード |
| ③ ブックマーク動機 | ①後で読み返したい（区別）／②誰かに送りたい（他者投影）／③自分のものにしたい（命名）／なし |
| ④ フックの構造 | 1行目が何をしているか（シーン・記憶・問い・逆説・命名） |
| ⑤ 未解決設計 | 「なぜか」「どうすればか」のどちらが空白か／両方書いて消費されていないか |
| ⑥ 数値との整合 | ブクマ率（ブクマ÷インプ）を計算。参考基準: 0.95%が過去分析の最高値 |

## 出力フォーマット（4部構成・毎回この順）

```
## 1. 投稿別分析表
（投稿ごとに観点①〜⑥の表）

## 2. 横断パターン
（複数投稿に共通する「伸びた理由」を最大3つ。1投稿だけの場合は省略）

## 3. MBTICODE変換案
（バズ構造を自アカウントの恋愛×タイプ文脈に変換した投稿案1〜2本。
 `brands/mbticode/sns_post_cheatsheet.md` の文体ルール準拠・`brands/mbticode/examples_sns.md` のGoodに寄せる）

## 4. 資産への反映提案
（cheatsheetに追記すべき新しい型・ルールがあるか。ある場合は追記文案を提示し、
 ユーザー承認後にのみ編集する。勝手に追記しない）
```

## 過去分析からの参照基準（判定のアンカー）

- 型αの参考: 「理由のある『好き』は、いつか冷める。／『なんか好き』は、一生狂う。」（68.8万インプ・ブクマ1,260）
- 採用しない型: 攻撃的ラベリング型・議論誘発型・エンタメ消費型（高インプでも低保存・転換ゼロのため分析対象から「学ばない」と明記する）

## 完了条件

出力4部構成がすべて埋まっていること。反映提案がない場合も「4. 反映提案: なし（既存ルールで説明可能）」と明記する。
