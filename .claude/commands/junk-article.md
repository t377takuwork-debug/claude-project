# junk juice 記事生成システム

/junk-themeの引き継ぎ情報を受けて、構成設計・本文生成・品質チェック・保存まで行うシステム。
引き継ぎ情報がない場合は「テーマ・ペルソナ・タイトル・種別を教えてください」と確認してから開始する。

## アカウント概要

- **バズ実績**：「1日3公演で沼落ちした話」いいね155・「ぴあアリーナ最高無敵の景色」いいね50
- **必須タグ**：#創作大賞2026 #エッセイ部門 #60代 #推し活
- **保存先**：`Junk314/junk_juice/articles/drafts/`

## Step 1：構成設計

### H2設計ルール

**感動・高揚型**
出会い・きっかけ → 没頭・深まり → ピーク体験 → 変化（ビフォーアフター）

**やらかし系**
状況設定 → やらかし発生 → 焦り・混乱（笑える解像度で） → ポジティブ転換で締める
※締めが弱いと読後感が暗くなる（いいね10記事の教訓）

**共通ルール**
- **冒頭はH2なしで本文から直接始める**（Noteエッセイは「つぶやきのように」入る）
- 各H2に「何のシーンを・どこまで具体的に書くか」の執筆方針を付ける
- H2は3〜5本に絞る

**H2見出しの言語設計**
- 全H2のうち最低1本に、読者の不安キーワード（コール・グッズ・一人・初めて等）または引き継ぎ情報の「H2組み込み候補ワード」を含める
- 純粋な感情語のみの見出し禁止（見出しだけでは何の話かわからないもの）
- 目標フォーマット：「[状況/キーワード] ＋ [感情・行動語]」
  例：「コールもグッズも知らないまま、1公演目へ」
- H2だけを縦に読んで記事の流れが追えるか、設計後に確認する

### H3の使用判断

体験エッセイは原則H3不要。1つのH2内に複数の独立した段階・情報があり、H3なしでは読者が迷子になる場合のみ使う。

## Step 2：本文生成

### 語り口のルール

- **一人称**：「私」（漢字。「わたし」は使わない）
- **テンポ**：短文と長文を交互に。段落間に空行を入れてリズムを作る
- **具体性**：「楽しかった」ではなく「あの瞬間、足が震えた」レベルの解像度で書く
- **捏造禁止**：実体験ベースのみ。不明な部分はユーザーに確認してから書く

---

### 文体参照集（published記事より抽出・確定表現）

本文生成時はこれらを文体の基準とする。

**【女性語・口語系語尾】**
- 「〜のよね。」「〜のよ。」「〜わ。」「〜わね。」「〜てね。」
- 「〜じゃん。」「〜じゃん！」
- 「〜んだよ。」「〜んだよね。」
- 「〜ちゃったの。」「〜しちゃったのよね。」

**【語尾の使用頻度ルール】**
- 「〜わ」「〜のよ」「〜のよね」は記事全体で各1〜2か所まで。感情ピーク・自己反省など「ここぞ」の瞬間にだけ使う
- 「〜んだよね」「〜じゃん」「〜の」は比較的自由に使えるが、同じ語尾を連続させない
- 口語語尾は「ここだけ本音が漏れた」という自然なリズムで配置する

**【驚き・感嘆表現】**
- 「ちょっと待って。」（単独1文）「マジで！？」「え？ えっ？」「は……？ え？」
- ！！連打は惜しまない

**【自己ツッコミ・ユーモアパターン】**
- 「のんきに〜していた」（間抜けさを笑う）
- 「〜とも知らずに。」（段落末の伏線・余韻）
- 括弧内の本音補足：「（オタク特有の幸せな錯覚なのかもしれないが！）」「（ビギナーではないな）」
- 「なんて、〜のです」（自分のセリフを「なんて」で受けてから転換）

**【接続・転換の定番ワード】**
- 「でもね、」（逆接の転換：最重要・最頻出）
- 「だって、聞いてよ。」（理由・詳細への誘導）
- 「そう、」「なんと、」「ところが、」「ふと」「気づけば」「そんなわけで、」

**【読者への語りかけパターン】**
- 「聞いてよ。」（単独文）「〜でしょ？」「ねえ、みんな〜ってしてる？」「〜くださいね！」

**【推し活固有語彙（自然に組み込む）】**
- 「沼にドボンと落ちた」「現場」「アドレナリンがブチ上がる」「全力で叫ぶ」「箱推し」「推しメン」

**【体言止め・短文の畳み掛け（感情ピーク時）】**
- 「気持ちいい。快感。」「情けない。申し訳ない。」
- ピーク直前に短文2〜3行を連続させてテンポを上げる

**【使わない表現】**
- 「ひとりごちた」→「つぶやいた」
- 「感慨深かった」（書き言葉すぎる）
- 過度な丁寧語一辺倒（〜でございます等）

### フォーマット・記号ルール

| 要素 | 使う場面 | 禁止 |
|---|---|---|
| `>` blockquote | 感情の山のセリフ・心の本音 | 通常の説明文・事実の羅列 |
| ` ``` ` コードブロック | 読者へのCTA・問いかけ | 地の文・説明文 |
| `**` 太字 | 無料記事：核心の一言（1〜2か所まで）／有料記事：鉄則セクションのヘッダーにも使用可 | 自己開示・通常の説明文 |
| `---` | 大きな転換点のセクション区切り | 多用しない（3〜4か所まで） |

### N（絞り込み）の定型（結論CTA前に必ず入れる）

```
この記事が全員に必要なわけじゃない。
推し活に興味がない人には関係ない話。
でも、行きたいのに踏み出せないでいる人がいるなら、読んでほしかった。
```
※テーマに合わせて文言を調整する。

### CTAのルール

- 「今夜」「今日」「今すぐ」など時間軸を入れる・行動は1つだけ・ハードルを低く（「まず」「〇〇だけ」）
- コードブロック ` ``` ` で囲む

### 記事末尾のフッター（必須）

```
---

junk juice｜60代のわくわくする余生の発信
Note：https://note.com/junk_juice
```

### 文字数目安

| 記事の型 | 目安 | 下限 |
|---|---|---|
| 通常無料記事 | 2,500〜3,500字 | 2,500字 |
| バズ狙い（感動体験記） | 3,500字以上 | 3,000字 |
| **有料記事（200円）** | **4,500〜5,500字** | **4,500字** |
| **有料記事（300円）** | **6,000〜7,000字** | **5,500字** |
| **有料記事（500円〜）** | **8,000字以上** | **7,000字** |

**有料記事の無料：有料比率（価格共通）**：無料3：有料7
- 無料パートが短すぎると有料転換が起きない。長すぎると満足されてしまう
- 計算例（200円）：合計5,000字なら無料1,400〜1,500字、有料3,500〜3,600字

## 有料記事モード（種別に「有料記事」が指定された場合のみ適用）

### 全体構造

有料記事は「無料パート」→「誘導文」→「ペイウォール」→「有料パート」の4層で構成する。

```
[無料パート] 読者を「欲しい」状態まで持ち込む
     ↓
[誘導文] 2〜3行・煽らない・何があるかを一言で告知
     ↓
[--- ※ここから有料パートです（Noteで公開時に有料ラインを設定する）]
     ↓
[有料パート] エピソード × 鉄則 × チェックリスト
```

### 無料パートの設計（Step 2 で適用）

**書くべき要素（この順番で）**：
1. **読者への問いかけ**：ペルソナの「夜も眠れない悩み」を代弁する（3〜5行）
2. **「私もそうだった」**：共感のブリッジ
3. **遠征前夜・挑戦前の不安描写**：具体的な情景で不安を1段落（200〜250字）
4. **「でも今は違う」→ビフォーアフター対比**：今できていることを具体的に列挙（150字）
5. **やらかし・失敗の概要**：3連発など簡潔に列挙（100字）
6. **ネガティブ→ポジティブの転換**：「やらかしが財産になった。その数だけ学べた」という転換（150字以内）
7. **この記事で何が解決するかの具体告知**：キーワードを列挙（150字）
8. **情景フック1文**：「失った何か」を情景で圧縮（例：「1曲目のイントロが外まで聴こえていた」）
9. **N（絞り込み）**：「〜に興味がない人には関係ない話。でも〜」（無料パートに置く）
10. **誘導文**：「[やらかし/テーマ]、全部これから話すね。[付加価値の一言]。続きをどうぞ。」

**情景フックの設計原則**：
- 「後悔の痛み」を長々と語らない
- 1文の情景描写で感情を引き起こす（「失ったもの」を音・景色・感覚で表現）
- ユーモアトーンを崩さない

**誘導文の型**：
```
[やらかし・テーマの概要]、全部これから話すね。[付加価値の一言]。続きをどうぞ。
```

### 有料パートの設計（Step 1 構成設計 + Step 2 本文生成 で適用）

**有料パートの型（やらかし系 → ノウハウ型昇華）**：
H2①〜③：[エピソード] → --- → [鉄則セクション]  の繰り返し
H2④（最終）：チェックリスト（最大の付加価値）

**鉄則セクションの書き方**：
- **ヘッダー**：`**鉄則① ◯◯する**` の太字（各H2に1本）
- **内容**：「何をすべきか」だけでなく「どうやってやるか」の具体手順まで書く
- **独自情報基準**：「ちょっと調べれば出てくる一般論」はNG。「経験者しか気づかない具体的手順」が必ず1点/H2必要
  - NG例：「Googleマップで確認する」
  - OK例：「Googleマップの乗換案内画面をスクリーンショット保存してオフラインで見る」
- **語調**：ですます調（鉄則セクションのみ。エピソード部分は普通体・口語を維持）

**文体2層設計**：
| セクション | 語調 | 語尾例 |
|---|---|---|
| エピソード部分 | 普通体 + junk_juice口語 | 〜のよ、〜わ、〜んだよね、〜だったわ |
| 鉄則・情報提供部分 | ですます調 | 〜します、〜ましょう、〜です、〜ありません |
| チェックリスト項目 | 普通体（完了形） | 〜した、〜する |

**--- の使用ルール（有料記事）**：
- 「エピソード → 鉄則」の切り替えシグナルとして使う（各H2に1か所）
- 鉄則 → 次のH2 の間は --- 不要（H2見出しが区切りになるため二重にしない）
- 合計3〜4か所（フッター前の --- を含む）

**チェックリストの品質基準**：
- 「あ、これ考えてなかった」と読者が思う項目のみ追加する
- 目安：15〜20項目（価格が上がるほど増やす）
- 各カテゴリに「実体験由来のひと言」を添える（「〜が実体験」「これ本当に大事」等）

## Step 3：品質チェック

本文生成後にチェック。問題があれば修正してから保存する。

**【資産性】**
- [ ] 読後にコメント・フォロー・シェアしたくなる余白があるか
- [ ] 「このアカウントの人が書いた」とわかる一次情報（実体験・固有シーン）が入っているか
- [ ] 「60代でもやれる」キャラクターが強化されているか
- [ ] 将来の有料記事（遠征マニュアル・沼落ち完全版）への導線になっているか

**【読みやすさ・拡散】**
- [ ] 最初の3行以内に共感フックがあるか
- [ ] 冒頭がH2なしで本文から始まっているか
- [ ] ありきたりな一般論で終わっていないか
- [ ] 「ライブ」「グッズ」「コール」「沼」「ハロプロ」等が自然に入っているか

**【フォーマット・スマホ縦読み】**
- [ ] `>` blockquoteが感情の山・本音に使われているか（説明文に使っていないか）
- [ ] blockquote前に助走段落（1〜2行の短文）があるか
- [ ] ` ``` ` コードブロックがCTAに使われているか
- [ ] `**` 太字が適切か（無料記事：1〜2か所のみ。有料記事：鉄則ヘッダーに追加可）
- [ ] N（絞り込み）が結論CTA前に入っているか
- [ ] H2見出しだけ縦読みで記事の流れが8割わかるか
- [ ] 1段落5行超が連続していないか（スマホで壁に見えない）
- [ ] 締め段落が「まとめ感」でなく「余韻・余白感」で終わっているか
- [ ] フッターが末尾にあるか
- [ ] 文字数が2,500字以上あるか

**【トーン】**
- [ ] ユーモア・自虐が1か所以上入っているか
- [ ] やらかし系の場合、締めがポジティブ転換になっているか
- [ ] 「深刻すぎる」「暗い」印象を与えていないか

**【有料記事 追加チェック】**（種別が有料記事の場合のみ）
- [ ] 無料パートに「失ったものを情景で表現した1文」が入っているか
- [ ] 無料パートに「ビフォーアフターの対比」が入っているか
- [ ] 無料パートに「この記事で何が解決するかの具体告知」が入っているか
- [ ] 誘導文が「煽らず・2〜3行で・何があるかを一言で」伝えているか
- [ ] 無料：有料の比率が 3:7 に近いか（PowerShellかPythonで文字数計測してから確認）
- [ ] 各鉄則に「一般論を超えた具体的手順・経験者の知恵」が1点以上入っているか
- [ ] 鉄則セクションがですます調、エピソード部分が普通体・口語になっているか
- [ ] エピソード部分にjunk_juice口語（〜のよ、〜わ、〜んだよね等）が入っているか
- [ ] チェックリストに「経験者しか気づかない項目」が含まれているか
- [ ] 総文字数が価格帯の下限を超えているか（200円→4,500字、300円→5,500字）
